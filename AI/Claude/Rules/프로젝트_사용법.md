📌 핵심 원칙 (Core Principles)

🎯 공통 템플릿 정의

Tier 시스템 (모든 해결책에 적용)

🟢 Tier 1: 즉시 해결 (1-3분, GUI 기반)
🟡 Tier 2: 기본 수정 (5-10분, 설정 변경)
🟠 Tier 3: 고급 조치 (30분+, CLI/레지스트리)
🔴 Tier 4: 시스템 재구성 (시간 변동, 재설치)


표준 체크리스트 구조

✅ 작업 전
- [ ] 환경 정보 수집 (OS/버전/하드웨어/소프트웨어)
- [ ] 백업/스냅샷 생성
- [ ] 소요시간 및 난이도 확인

✅ 작업 중
- [ ] 단계별 실행 및 기록
- [ ] 예상 결과와 실제 결과 비교

✅ 작업 후
- [ ] 기능 정상 작동 검증
- [ ] 부작용 확인
- [ ] 롤백 방법 숙지


정보 표시 형식

⭐ 난이도: ⭐~⭐⭐⭐⭐⭐
📊 성공률: XX%

도구 전환 3단계

Phase 1: 병렬 운영 (1-2주)
- 기존 도구 + 신규 도구 동시 사용
- 동일 작업을 양쪽에서 수행하여 비교

Phase 2: 점진적 전환 (2-4주)
- 새로운 작업은 신규 도구 우선
- 기존 작업은 기존 도구 유지

Phase 3: 완전 전환 (1주)
- 모든 작업을 신규 도구로 이관
- 기존 도구는 백업용으로만 유지

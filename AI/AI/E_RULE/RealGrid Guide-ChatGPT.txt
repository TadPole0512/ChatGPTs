## **D. Easy-to-confuse Essentials (with official docs)**

* Overview

  * ‚ñ∑ Objects / Order

    * Create **LocalDataProvider** (data) and **GridView** (view), connect with `gridView.setDataSource(provider)` ‚Üí define fields/columns ‚Üí set options/callbacks ‚Üí call `provider.setRows(data)` (recommended order). ([docs.realgrid.com][1])
  * ‚ñ∑ Install / License

    * NPM/CDN/download are all possible; **license is required**. Use a license script file or `RealGrid.setLicenseKey()`. ([docs.realgrid.com][1])
  * ‚ñ∑ Sorting / Filtering

    * Sorting: `gridView.sortingOptions.enabled` / multi-sort style (`exclusive/inclusive`, etc.). ([docs.realgrid.com][2])
    * Filtering: `gridView.setColumnFilters()`, `FilteringOptions.enabled`, etc., to configure/toggle/hide per-column filters. ([docs.realgrid.com][3])
  * ‚ñ∑ Excel Export

    * Use `gridView.exportGrid({ type:"excel", ... })`; **jszip** is required (supports local/remote modes). ([docs.realgrid.com][4])

> **Key takeaway: First fix ‚ÄúProvider‚ÜîGridView wiring‚Äù and the ‚Äúsorting/filtering/export core APIs.‚Äù**

## **F. Server Delegation Policy (sorting/filtering/paging)**

* Overview

  * ‚ñ∑ Why?

    * With large datasets, doing filter/sort/page purely on the client can cause performance issues.
* How to use

  * ‚ñ∑ Procedure

    * ‚Üí Consider server delegation beyond a **size threshold** (e.g., ‚â• 50k rows).
    * ‚Üí Define a mapping table (column ‚Üí API params/operators).
    * ‚Üí Grid events (filter/sort changes) ‚Üí re-query server ‚Üí apply via `provider.fillJsonData()` or `setRows()`.
* Note: RealGrid provides toggles/styles/condition-based filters; when delegating, pass only the **state** to the server. ([docs.realgrid.com][2])

> **Key takeaway:** First create a table that **translates grid state into server params**.

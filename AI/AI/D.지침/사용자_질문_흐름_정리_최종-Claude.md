사용자가 제공한 5개 문서를 분석해보니, 주로 AI(ChatGPT/Claude) 사용 시 주의사항과 질문 처리 지침에 관한 내용들입니다. 중복되는 내용들을 제거하고 체계적으로 정리하여 하나의 통합된 가이드라인을 만들어보겠습니다.

**Genius Insight Formula (GI)** 적용:
- **관찰(O)**: 문서들 간 중복 내용과 핵심 원칙 파악
- **연결(C)**: 분산된 내용들의 논리적 연관관계 구성
- **패턴(P)**: 사용자 질문 → 처리 → 답변의 일관된 흐름 인식
- **종합(S)**: 실용적이고 체계적인 통합 가이드라인 구성

# 🤖 AI 활용 및 질문 처리 통합 가이드라인

## 📋 **1. AI 사용 시 기본 원칙**

### ✅ **반드시 확인할 것**
- 개인정보(실명, 주민번호, 계좌번호, 비밀번호 등) 입력 금지
- 의료, 법률, 금융 등 중요한 결정은 전문가와 확인
- 코드/분석/문서 결과는 직접 검증 및 백업 필수
- 저작권/상표권 등 상업적 사용 전 라이선스 확인
- 최근 대화 640개 내에서만 맥락 기억, 장기기억은 직접 요청 필요

### 🚫 **절대 금지사항**
- 민감한 개인정보 입력
- 불법, 해킹, 범죄 관련 질문
- 차별, 혐오, 폭력, 음란 등 비윤리적 요청
- 실제 행동(결제, 로그인, 기기 조작 등) 대행 요청
- 중요 결정을 AI 답변만으로 바로 실행

---

## 🎯 **2. 효과적인 AI 활용 방법**

### **구체적 질문 원칙**
- 원하는 방식/포맷/결과를 처음부터 명확히 제시
- "비교", "대안", "다른 방식도 알려줘" 같은 시야 확장 표현 활용
- 포맷, 말투, 출력방식 등은 한 번 정하면 계속 적용

### **지시사항 관리**
- 질문/지시가 바뀔 때는 새 기준을 명확히 전달
- "앞으로는 ○○ 방식으로", "이번엔 △△로" 등 명시적 지시
- 중요하거나 반복할 기준은 "기억해달라"고 장기기억 요청

### **관점 다양화**
- 답변이 한 방향으로만 흐르면 "다른 관점/방법" 요청
- 충돌이나 모순 의심 시 직접 확인 요청
- "내가 생각 못 한 대안도 제안해줘" 활용

---

## 🔄 **3. 질문 처리 단계별 절차**

### **1단계: 사용자 의도 분석**
- 전체 입력을 문장 단위로 분석
- **'사전 설명(상황 설명)'**과 **'질문 자체'**로 구분
- 여러 내용이 있을 경우 연관성에 따라 같은 유형끼리 묶기
- 질문들의 선후 처리 순서 파악

### **2단계: 질문 구조화**
- `사전 설명 → 질문들` 단위로 묶어서 정리
- 복잡하거나 논쟁 여지가 있는 경우만 사용자에게 확인
- 사용자 수정 요청 시 내용 반영하여 재정리

### **3단계: 단계별 답변 처리**
- 묶인 `사전 설명 + 질문들`을 하나씩 단계별로 처리
- 사전 설명을 바탕으로 질문에 대한 답변 제시
- **절대 생략하지 말고 전체 답변 출력**
- 내용이 길 경우 여러 번에 나누어 전체 답변 화면 출력

### **4단계: 진행 관리**
- 다음 단계로 넘어갈지 여부 확인
- 완료된 단계는 번호를 매겨 정리
- 모든 단계 완료 시 전체 내용을 최종 결과로 전달

---

## ⚠️ **4. 특수 상황 대응 지침**

### **반복 질문 감지 및 대응**
- 동일하거나 유사한 질문이 **5회 이상 반복** 시 감지
- 반복 감지 시 즉시 사용자에게 알림
- 설명 방식 변경 제안 (표, 리스트, 실제 예시, 스토리텔링, 비유)
- 질문 방향 전환 또는 역질문 유도

**대응 예시 문장:**
- "같은 주제로 5번 이상 반복된 질문이에요. 혹시 설명 방식을 바꿔볼까요?"
- "지금까지와 비슷한 질문인데, 다른 관점이나 표현으로 다시 정리해볼까요?"

### **대화 흐름 관리**
- 대화 흐름을 끊거나 이어갈 때 명확한 신호 제공
- "지금 내용 마무리하고 새 주제로", "이전 이야기 이어서 계속" 등
- 대화 길이가 길어질 때 "요약"이나 "중간 정리" 요청 가능

### **답변 길이 조절**
- 답변이 길어질 경우 대략적인 흐름과 요약된 결과를 먼저 안내
- 이후 상세 내용을 단계별로 나누어 제공
- 사용자가 원하는 형식으로 재요청 가능 ("완전히 다르게", "짧게/길게/표로")

---

## 🛠️ **5. 실행 환경 및 기술적 요구사항**

### **검증 원칙 (모든 답변에 적용)**
- **URL/링크**: HTTP 상태 코드 확인, 절대 경로 변환
- **숫자/날짜**: 수식과 입출력 값 제시, 절대값으로 기록
- **코드/SQL**: 2개 샘플 입력 + 예상 출력 + 1개 오류 케이스 포함
- **증명**: CSV/로그/소스를 결과와 함께 제시

### **언어 및 출력 기준**
- **기본 언어**: 한국어 (메뉴명, 용어, 코드 설명 포함)
- **파일 형식**: 원하는 형식/스타일 명확히 요구 (.md, csv, 탭 들여쓰기 등)
- **완전성**: 부분 출력 금지, 전체 내용을 나누어서라도 완전 출력

### **금지 패턴**
- "모른다면 전문가에게", "상황에 따라 다름"(구체적 분석 없이)
- "잘 모르겠다", "전문가가 아니라서" 등의 회피성 답변
- 추측성 표현 ("아마도", "보통") 사용 금지

### **사용자 경험 최적화**

- [ ] 전체 내용 생략 없이 완전 출력

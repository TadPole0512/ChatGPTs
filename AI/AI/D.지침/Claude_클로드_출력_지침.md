# 🎨 클로드 AI 스타일 출력 지침 작성법

## 📋 핵심 원칙

### 1. 계층적 구조화 (Hierarchical Organization)
```markdown
# 대제목 (H1) - 주제 전체를 포괄
## 중제목 (H2) - 주요 섹션
### 소제목 (H3) - 세부 항목
#### 세세한 제목 (H4) - 구체적 내용
```

### 2. 시각적 구분자 활용
- **굵은 텍스트**: 핵심 키워드 강조
- *기울임 텍스트*: 부연 설명이나 예시
- `코드 블록`: 기술적 용어나 명령어
- > 인용문: 중요한 포인트나 주의사항

### 3. 이모지 전략적 배치
- 📌 **섹션 헤더**: 내용의 성격을 직관적으로 표현
- ✅ **체크리스트**: 완료 항목이나 권장사항
- ⚠️ **주의사항**: 경고나 중요한 알림
- 💡 **팁**: 추가 정보나 아이디어

## 🎯 구체적 적용 방법

### A. 문서 시작 구조
```markdown
# 🎯 [메인 주제]

## 📋 개요
- **목적**: 명확한 목적 설명
- **범위**: 다룰 내용의 범위
- **대상**: 주요 대상 독자

## 🔍 핵심 내용
```

### B. 내용 전개 패턴
1. **문제 정의** → 2. **해결 방안** → 3. **실행 단계** → 4. **검증 방법**

### C. 리스트 구조화
```markdown
### 📝 방법 1: 기본 접근법
- **장점**: 
  - 간단하고 직관적
  - 빠른 구현 가능
- **단점**:
  - 확장성 제한
  - 커스터마이징 어려움

### 🚀 방법 2: 고급 접근법  
- **장점**:
  - 높은 유연성
  - 전문적 결과물
- **단점**:
  - 학습 곡선 존재
  - 초기 설정 복잡
```

## 🎯 맞춤형 지침 3가지 옵션

### 🚀 옵션 1: 기술 문서용
- 코드 블록 40%, 설명 텍스트 60%
- 단계별 번호 매기기 강조
- 에러 케이스와 해결법 포함

### 📊 옵션 2: 리포트용  
- 데이터 시각화 30%, 분석 텍스트 70%
- 표와 차트 적극 활용
- 핵심 인사이트 박스 처리

### 🎨 옵션 3: 가이드용
- 스크린샷 20%, 설명 80%  
- 체크리스트와 실행 단계 중심
- 초보자 친화적 언어 사용

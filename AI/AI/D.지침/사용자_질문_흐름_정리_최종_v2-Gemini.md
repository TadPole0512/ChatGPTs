## 🛠️ Part 2. 복잡한 질문 처리 프로세스 (AI 시스템 로직)

AI가 사용자의 **복잡한 입력**을 받았을 때, 답변 누락 없이 체계적으로 처리하고 전달하는 **시스템의 내부 작업 흐름**입니다.

### 2.1. 🧠 입력 분석 모듈 (Input Analysis)

1.  **🔍 문장 단위 구분:** 전체 입력 내용을 '사전 설명(상황 설명)'과 '질문 그 자체'로 분리합니다.
2.  **🔗 유형별 묶음 처리:** 사전 설명과 질문을 연관성에 따라 **같은 유형끼리 묶어** `사전 설명 → 질문들` 단위로 구성합니다.
3.  **🔢 순서 정리:** 질문들의 흐름을 파악하여 **선후 처리 순서**를 확정합니다.

### 2.2. ⚙️ 단계별 실행 로직 (Step-by-Step Execution)

| **단계** | **프로세스** | **주의사항 (🚨 Alert)** |
| :--- | :--- | :--- |
| **Step 1** | **묶음 처리 진행** | 묶인 `사전 설명 + 질문들`을 순서대로 하나씩 처리 시작 |
| **Step 2** | **확인/보정** | 복잡하거나 논쟁 여지가 있는 경우, 현 단계 내용 (**사전 설명과 질문**)을 사용자에게 **재정리하여 확인**받고 **예시를 요구**함. |
| **Step 3** | **피드백 반영** | 사용자가 수정/추가 요청 시, 해당 내용을 반영하여 **다시 정리**해 보여줌. |
| **Step 4** | **답변 출력** | 진행 승인 시, 사전 설명을 바탕으로 답변 제시. <br> **🚨 [출력 보장]** 답변은 생략 없이 전체를 출력하며, 길 경우 **반드시 여러 번에 나눠** 화면에 출력함. |
| **Step 5** | **다음 단계 문의** | 현 단계 완료 후, 사용자에게 **다음 단계 진행 여부**를 문의함. |
| **Step 6** | **단계 정리** | 완료된 단계는 `사전 설명 + 질문 + 답변`을 묶어 **번호 매겨 정리**하고 다음 단계로 이동. |
| **Step 7** | **최종 전달** | 모든 단계 완료 시, 전체 내용을 **생략 없이 흐름 순으로 정리하여** 사용자에게 최종 전달함. |

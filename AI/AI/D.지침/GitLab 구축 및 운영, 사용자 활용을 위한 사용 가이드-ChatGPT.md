# 🧭 공통 메타 프롬프트 (처음에 꼭 한 번)

1. **문서 뼈대(TEMPLATE)**

* “GitLab 가이드의 전체 목차와 산출물 형식을 잡아줘. 영역은 **구축·운영·사용자 활용**으로 나누고, 각 섹션에 **목표/전제/절차/검증/트러블슈팅**을 포함해. 표·체크리스트·코드블록 중심으로 **한 번에 복사해 쓰기 좋은 형식**으로 만들어줘.”

2. **전제 명시 & 버전 고정**

* “이 문서는 **GitLab ⟨버전⟩**, **OS: ⟨배포판/버전⟩**, 사용자 수 ⟨N⟩, 동시 사용자 비율 ⟨X%⟩, 네트워크 제약 ⟨설명⟩을 전제로 해. 모든 명령과 설정은 이 전제를 기준으로 **재현 가능**하게 적어줘.”

3. **산출물 형태 고정**

* “각 절차는 ‘**Before–Action–Verify**’ 3단계로, 마지막에 **검증 명령/기대 결과**와 **실패 시 원인/해결**을 표로 달아줘. 가능하면 **스크립트/샘플 파일**도 함께 제공해줘.”

# 🏗️ 구축(Installation/Architecture) — 질문 세트

1. **아키텍처 결정**

* “우리 상황(사용자 ⟨N⟩, 동시성 ⟨X%⟩, 예산 ⟨범위⟩, 가용성 목표 ⟨SLA/SLO⟩)에서 **Omnibus / Docker / Kubernetes(Helm) / 클라우드 매니지드** 중 **의사결정 트리**와 **권장안**을 표로 제시하고, 이유/장단점/리스크/확장 경로를 정리해줘.”

2. **사이징 가이드**

* “⟨N⟩명 기준 GitLab 단일/HA 구성의 **CPU/RAM/스토리지/IOPS/네트워크** 권장 스펙과 근거를 공식 지표/가정으로 산출하고, **소형/중형/대형** 3옵션 표를 만들어줘.”

3. **Omnibus 설치 가이드(예시)**

* “Ubuntu ⟨버전⟩ 기준 **명령어만 복붙하면 되는** Omnibus 설치 스크립트와, `gitlab.rb` 최소 설정 템플릿, **HTTPS/방화벽/백업 기본값** 포함. 설치 후 **스모크 테스트** 5개와 기대 출력까지.”

4. **Kubernetes(Helm) 배포**

* “Helm 차트 버전 ⟨v⟩ 기준 **values.yaml 템플릿**을 만들고, **Ingress/스토리지/Runner/레지스트리/모니터링** 옵션을 주석으로 설명해줘. `helm upgrade --install` 명령과 **롤백 절차**도 포함.”

5. **보안 초기 하드닝**

* “초기 구축 시 필수 보안 설정(HTTPS 강제, SSH/포트 정책, 관리자 계정/비밀번호 정책, 감사 로그, 비밀관리) **체크리스트 + 코드/설정 예시**를 제공해줘.”

6. **인증 연동**

* “회사 **LDAP/SAML/OIDC**와의 연동 절차를 **사전 체크 → 설정 → 검증** 3단계로, 설정 스니펫과 **테스트 시나리오**까지.”

7. **컨테이너·패키지 레지스트리**

* “Container/Package Registry 활성화 방법, **용량/보존 정책**, 퍼지(purge) 전략과 자동화 예시(CRON/GitLab 스케줄러).”

# 🔧 운영(Operations) — 질문 세트

1. **백업/복구 전략**

* “RPO ⟨분⟩ / RTO ⟨분⟩ 목표로 **일일 자동 백업 스크립트**, 보관/암호화/원격 업로드(S3 등) 설정, **복구 리허설 체크리스트**와 **복구 시간 측정 방법**을 만들어줘.”

2. **모니터링 & 알림**

* “Prometheus/Grafana 기준 **대시보드 지표 목록(SLI)**, 임계치/알림 규칙(YAML), **헬스체크 엔드포인트**와 점검 절차를 제공해줘. ‘장애 감지 → triage → 에스컬레이션’ **런북** 포함.”

3. **업그레이드/롤백**

* “버전 ⟨from→to⟩ 업그레이드 **사전 점검 목록**, 절차(무중단 옵션 포함), **롤백 절차**와 데이터 마이그레이션 주의사항을 Before–Action–Verify로.”

4. **성능/용량 관리**

* “레포/아티팩트/레지스트리 성장률 가정하에 **용량 예측 표**, 성능 병목(스토리지/DB/Gitaly/네트워크) **진단 명령 모음**과 튜닝 권장값.”

5. **장애 대응(Incident)**

* “상위 10개 **빈발 이슈**(502/Runner 대기/CI 큐 정체/레지스트리 오류 등)의 **징후→원인→해결→재발 방지**를 표로 정리하고, 각 케이스의 **로그 위치/명령/타임라인 템플릿**을 줘.”

6. **권한/라이선스 거버넌스**

* “그룹/프로젝트 **권한 매트릭스**(원칙/예외), 계정 라이프사이클 온·오프보딩 체크리스트, 라이선스 사용량 **리포트 생성 스크립트**.”

7. **감사/컴플라이언스**

* “변경 이력, Merge Request 규칙, 승인 정책, 감사 로그 보존 정책을 **감사 대응용 체크리스트**로 만들어줘.”

# 👥 사용자 활용(Enablement) — 질문 세트

1. **프로젝트 구조/템플릿**

* “모노레포/멀티레포 선택 기준과 **추천 구조**(그룹/서브그룹/프로젝트), **Issue/MR 템플릿** 마크다운 샘플을 생성해줘.”

2. **Git 워크플로**

* “우리 팀 특성(배포 빈도 ⟨n/주⟩, 협업 방식 ⟨코드오너/리뷰어⟩)에 맞춘 **브랜치 전략(Git Flow/GitHub Flow/Trunk)** 비교표와 **MR 규칙**(라벨/승인수/체크) 제안.”

3. **CI/CD 템플릿 자동 생성**

* “⟨언어/프레임워크⟩용 **.gitlab-ci.yml**을 ‘초급(빌드만)→중급(빌드+테스트)→고급(보안 스캔+배포)’ 단계별로 만들어줘. **캐시/아티팩트/병렬/매트릭스** 예시 포함.”

4. **배포 전략**

* “스테이징/프로덕션 분리, **Blue-Green/Canary/Rolling** 각각의 **파이프라인 예시**와 **환경 변수/시크릿 관리** 모범사례를 제시해줘.”

5. **보안 스캐닝**

* “SAST/DAST/Dependency/Container 스캔을 **파이프라인에 통합**하는 예시와, **임계값/게이팅 정책**(fail 조건) 및 **리포트 해석 가이드**를 작성해줘.”

6. **Runner 전략**

* “공유형 vs 전용형 **Runner 설계**, 오토스케일링(예: Docker Machine/K8s) 방법, **태그 정책**과 **대기 시간 줄이기** 팁을 정리해줘.”

7. **릴리스 관리**

* “버전네이밍, 체인지로그 자동화, 릴리스 태그/패키징/아티팩트 업로드 **파이프라인**과 **릴리스 체크리스트**를 만들어줘.”

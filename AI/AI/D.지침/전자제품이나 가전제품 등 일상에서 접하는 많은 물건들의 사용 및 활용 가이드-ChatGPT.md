# 🧰 전자·가전제품 “정확한 대답”을 위한 사전 지침 v1.0

## 🧭 개요 (Overview)

* **목적**
  일상에서 쓰는 전자·가전제품(예: TV, 냉장고, 로봇청소기, 공유기, 헤드폰 등)의 **사용·활용을 극대화**하기 위해, 질문 전에 **반드시 고정**해야 할 정보·증거·검증 절차를 **GUI 중심**으로 표준화.
* **적용 범위**
  구매 전 비교, 설치/연결, 기능 최적화, 고장 진단, 유지보수/펌웨어 업데이트, 스마트홈 연동.
* **대상**
  초보자~파워유저, 가정/소규모 사무 환경(Windows, iOS/Android, Wi-Fi/BT/IoT)

> **Key takeaway: “환경 고정 → 증거 수집 → 문제유형 분류 → 단계별 검증 → 최종 요약” 순서를 항상 지킨다.**

## 🧩 A. 프레이밍 & 원칙

* Overview

  * ▷ Background

    * 가전 문제의 70%는 **환경/버전/연결** 미고정으로 발생
    * 모델명/펌웨어/앱 버전/지역 규격/전원/네트워크 정보가 핵심 변수
  * ▷ 원칙

    * **GUI 우선**: “어디서 무엇을 클릭했는지” 경로 기록
    * **재현 가능성**: 같은 절차로 같은 결과가 나는지
    * **증거 중심**: 스크린샷·사진·로그·영상 20초 이내

> **Key takeaway: 말보다 ‘화면·사진’이 먼저다.**

## 🧾 B. “한 번에 정확한 답”을 위한 사전 템플릿 (붙여넣기용)

### 1) 제품 카드(필수)

* **제품명/모델명**: (예: Samsung QN85B, Dyson V12, Sony WH-1000XM5)
* **제조년/리전**: (예: 2023, KR/EU/US)
* **펌웨어/앱 버전**: 본체 v__, 모바일 앱 v__
* **전원/규격**: 220V/110V, 주파수, 어댑터 정격
* **연결**: Wi-Fi 2.4/5GHz, BT 버전, Zigbee/Thread/Matter 유무
* **연동 기기**: 스마트폰 OS/버전, 라우터 모델/펌웨어, 허브(있다면)

### 2) 환경 카드(필수)

* **설치 위치**: 벽면/캐비닛/욕실/베란다(온·습도, 환기)
* **거리/간섭**: 라우터까지 m, 전자레인지/두꺼비집 근접?
* **네트워크**: SSID, 암호방식(WPA2/WPA3), DHCP/고정 IP, DNS
* **전원 품질**: 멀티탭/서지보호/정전 이력

### 3) 증거 카드(필수)

* **라벨 사진**: 모델/시리얼/전력 라벨 근접샷
* **앱 스크린샷**: 오류코드/설정 경로(예: “앱 > 설정 > 펌웨어 업데이트”)
* **동영상(선택)**: 재현 영상 10~20초(문제 발생 순간 포함)
* **로그/시간**: “2025-09-30 20:15 KST”처럼 절대시간 표기

### 4) 요구사항 카드(선택)

* **목표**: (예: “eARC 끊김 없이 5.1 유지”, “청소 구역 스케줄”)
* **제약**: 소음/전력/예산/미관/가구 배치
* **평가 기준**: 성공 판정 조건(예: 1주간 끊김 0회)

> **Key takeaway: 위 4장(제품/환경/증거/요구)을 복붙해 채우면 답 정확도가 급상승한다.**

## 🧭 C. 문제 유형별 “질문 트리” (GUI 경로 포함)

### 1) 설치·연결

* ▷ 증상 분류

  * Wi-Fi 등록 실패 / BT 페어링 실패 / 앱 장치 탐색 불가
* ▷ 절차

  * → 라우터: **관리자 페이지**(예: 192.168.0.1) 접속 → 2.4/5GHz SSID 분리 여부·WPA 모드 확인
  * → 스마트폰: 설정 > Wi-Fi 상세 > IP/DNS/프라이빗 주소 스냅샷
  * → 장치: 초기화(Reset 버튼 ___초), 앱 경로: “앱 > 기기 추가 > 모델 선택 > 안내 팝업 캡처”
* ▷ 증거

  * 실패 팝업 전체화면, 라우터 무선 설정 화면

### 2) 기능·성능 최적화

* ▷ 예시

  * TV 모션/로컬디밍/게임모드, 공기청정기 자동모드 임계, 로봇청소기 지도 관리
* ▷ 절차

  * → “메뉴 > 영상(또는 장치 옵션) > 고급” 경로 스샷
  * → 조도/거리/장애물 사진
  * → 전/후 비교(표준 테스트 10분·동일 콘텐츠)

### 3) 소리·화질·연결끊김

* ▷ 예시

  * eARC/CEC 이상, BT 딜레이, Wi-Fi 스루풋 저하
* ▷ 절차

  * → 소스기기(콘솔/셋톱) 출력 포맷 화면
  * → TV/사운드바 CEC/eARC 토글 화면
  * → Speedtest 결과(2.4/5GHz 각각)

### 4) 충전/배터리/발열/전력

* ▷ 절차

  * → 충전기·케이블 정격 라벨 사진
  * → 소비전력 측정(스마트플러그 앱 스샷)
  * → 통풍/방열 사진

### 5) 오류코드/알림/이상동작

* ▷ 절차

  * → **오류코드 원문 그대로** 캡처
  * → 발생 빈도·시간대 기록(절대시간 표기)
  * → 마지막 성공 동작 시점/변경사항(업데이트/이동/베타앱)

> **Key takeaway: “경로·토글·숫자”가 보이는 스크린샷이면 해결 속도가 2배.**

## 🔍 D. 검증(Validation) & 재현(Repeatability)

* Overview

  * ▷ 목적: “운 좋게 됐다”가 아니라 **누구나 따라하면 된다**를 보장
  * → 테스트 시나리오 3종: **정상/경계/장애**
* **표준 검증표(복붙용)**

  * 항목 | 절차 | 기대값 | 결과(성공/실패) | 스크린샷 링크
  * Wi-Fi 재연결 | 앱>기기재검색 | 30초 내 연결 | □/□ | [ ]
  * 펌웨어 확인 | 설정>정보>버전 | 최신 v__ 표기 | □/□ | [ ]
  * 기능 A | 경로: … | 지연 ≤ 100ms | □/□ | [ ]
* **성공 기준(예)**

  * 1주간 중단 0회, 지연 중앙값 ≤ 80ms, 에러코드 미발생

> **Key takeaway: “테스트 표 + 스크린샷 링크”가 최종 답의 신뢰도다.**

## 🛡️ E. 안전·보증·AS 체크

* Overview

  * ▷ 안전

    * 누전·발열·물이 닿는 환경 주의, **정격 초과 어댑터 금지**
  * ▷ 보증

    * 자가 분해·비공식 펌웨어·전압변환 사용 시 보증 무효 가능
  * ▷ AS

    * **시리얼/구매영수증/문제 재현 영상**을 하나의 폴더로 정리 후 접수

> **Key takeaway: 안전·보증을 위반하는 설정은 ‘안된다’. 증거 모아 AS를 이긴다.**

## 🧠 F. AI에게 물을 때 쓰는 “정확한 질문 프롬프트” (복붙)

### 1) 설치·연결용

```
[제품/모델] [펌웨어 v__], [앱 v__], [스마트폰 OS], [라우터 모델/암호방식].
증상: [예: 2.4GHz만 보이고 5GHz 미표시].
시도한 절차: [앱 > 기기추가 > ...], [라우터 채널 36/149 교체], [재부팅].
증거: [오류 팝업 스샷], [라우터 무선설정 스샷].
목표: [예: 5GHz로 안정 연결].
제약: [예: SSID 분리 불가].
“GUI 경로 기반 단계별 해결책과 각 단계 성공 판정 기준을 표로 주세요.”
```

### 2) 최적화/활용용

```
[제품/모델] 사용 목적: [영화/게임/회의/수면].
환경: [조도/거리/방 위치/흡음].
현재 설정 스샷: [영상/음향/게임모드].
목표 지표: [밝기/색정확도/지연/자율주행 성공률].
“프로필 3개(기본/야간/게임)로 GUI 경로와 수치 제안 + 검증 절차를 주세요.”
```

### 3) 오류코드 진단용

```
오류코드: [정확한 문자열].
발생 시점/빈도: [절대시간/횟수].
변경 이력: [펌웨어/이동/신규 IoT].
“가능 원인별(상/중/하) 우선순위, 재현 스텝, 스크린샷 포인트를 카드로 정리해주세요.”
```

> **Key takeaway: “증상·시도·증거·목표·제약” 5요소가 정확도 핵심.**

## 🧰 G. 장치군별 핵심 체크리스트(요약)

| 장치군     | 최우선 확인                  | GUI 경로 힌트           | 흔한 원인             |
| ------- | ----------------------- | ------------------- | ----------------- |
| TV/사운드바 | eARC/CEC, 입력 포맷         | 설정>연결>HDMI-CEC/eARC | 케이블 규격/CEC 충돌     |
| 공유기/메시  | SSID 2.4/5 분리, 채널·대역폭   | 무선설정>고급             | DFS 채널·WPA 모드 불일치 |
| 로봇청소기   | 지도/금지구역, 브러시 마모         | 앱>지도관리              | 조도·문턱, 펌웨어 구버전    |
| 에어컨/공청기 | 자동모드 임계, 필터 수명          | 앱>장치설정>자동           | 센서 오염·흡기/배기 막힘    |
| 이어폰/헤드폰 | 코덱(SBC/AAC/LDAC), 멀티포인트 | 블루투스>코덱             | 2.4GHz 간섭·펌웨어불일치  |
| 프린터     | 드라이버/스풀러/2.4GHz         | 제어판>프린터속성           | 5GHz 미호환·WPS 실패   |

> **Key takeaway: 장치군마다 “한두 개 토글+케이블/채널”이 승부처.**

## 🧱 H. 문서화 & 공유 규격 (GUI 스타일)

* 카드 레이아웃: **문제 요약卡 / 환경卡 / 증거卡 / 해결案卡 / 검증표**
* 표기 규칙: 절대시간(KST), 버전 fully qualified 표기(예: v1.2.3-build45)
* 파일 구조: `device_model/YYYYMMDD_issue/` (스크린샷·영상·로그 정리)
* 결과물: “요약 카드(3줄) → 단계별 스텝(숫자) → 검증표 → 부록(사진 링크)”

> **Key takeaway: 누가 봐도 바로 실행 가능한 한 화면 요약.**

## ✅ 체크리스트

1. 제품/환경/증거/요구 4카드 모두 채웠는가
2. GUI 경로가 **화면 글자 그대로** 적혔는가
3. 오류코드 **원문 그대로** 캡처했는가
4. 절대시간·버전 표기를 썼는가
5. 테스트 **정상/경계/장애** 3종을 수행했는가
6. 전/후 비교 수치 또는 체감 영상이 있는가
7. 보증·안전 위반 요소가 없는가
8. 최종 **한 화면 요약**이 있는가

> **Note:** 실패를 피하려면 “증거가 없는 서술”을 금지하고, 각 단계에 **성공 판정 기준**을 붙인다.

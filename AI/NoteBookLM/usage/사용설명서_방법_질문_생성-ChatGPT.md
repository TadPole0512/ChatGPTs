# 📌 NotebookLM 전기능 사용설명서 — 복붙용 마스터 프롬프트

```
[목표]
나는 NotebookLM을 처음 쓰는 사용자야. NotebookLM의 "모든 기능"을 초등학생도 이해할 수 있을 정도로 쉽고, 화면 중심(시각 효과 극대화)으로 설명해줘. 화면 출력이 너무 길어지면 마지막에 전체 내용을 .md 및 .pdf 파일로도 만들어서 다운로드 경로를 제공해줘.

[범위]
- 기본 개념 → 계정/접속 → 새 노트북 만들기 → 자료(문서/웹/Drive) 추가 → 소스 정리·태깅
- Q&A/브리프 생성(Notebook Guide/Source-Grounded) → 인용/근거 표시 → 노트·요약·하이라이트
- 오디오 브리프/스튜디오(있다면: 스크립트 만들기, 장면 구성, 미리보기, 내보내기)
- 협업(공유/권한) → 히스토리/버전 → 검색/고급 검색 → 설정/개인정보 → 한계/주의사항
- 고급 활용: 학습세트 만들기, 시험문제 생성, 회의 요약, 리서치 리포트/블로그 초안, 프롬프트 패턴 라이브러리

[출력 형식 (GUI 우선)]
1) 가장 위에 "📚 목차" 카드와 진행 뱃지(기초/중급/고급)
2) 각 기능은 카드형 섹션으로:
   - 🎯 한줄 요약
   - 🛠 준비물/전제(계정, 권한, 파일 형식 등) 체크리스트
   - 🔄 따라하기(클릭 경로를 화살표로: 예) 홈 ▶ New Notebook ▶ Add sources ▶ …)
   - 👀 화면 미리보기 설명(버튼/아이콘 위치, 틀린 사례 스샷 설명 대체 텍스트)
   - ✅ 완료 점검(3~5개 체크)
   - ⚠️ 흔한 실수 & 해결법(원인→해결 1줄)
   - 💡 응용/고급 팁(프롬프트 예시 2~3개)
3) 표는 3~6열, 10행 이내(필요 시 표를 나눠서)
4) 중요 경고는 붉은 경고 상자, 참고는 파란 정보 상자
5) 단계가 긴 파트는 "자세히 보기" 접기/펼치기(아코디언)로 나눠서 출력
6) 마지막에 "전체 요약 카드" + "학습 로드맵(1주 코스)" + "프롬프트 치트시트" 포함

[톤 & 난이도]
- 초등학생도 이해: 어려운 말 금지, 쉬운 비유(“노트북=폴더, 소스=책/자료”) 사용
- 한국어 기본, 영어 메뉴명은 괄호 병기 (예: 소스 추가(Add sources))
- 단계는 번호로, 클릭/입력은 ■ 굵게 표시

[검증/근거]
- 각 기능 설명에 실제 버튼/메뉴 경로를 굵게 인라인 표기
- 기능 사용 예시는 2가지 이상(학습/업무)
- 중요한 한계(저작권, 개인정보, 환각 가능성)와 회피법 명시

[샘플 시나리오(필수 포함)]
- 시나리오 A: 수업 자료로 과학 챕터 3개 업로드 → 요약노트 만들기 → 퀴즈 5문항 자동 생성
- 시나리오 B: 회의록 2개 + 웹페이지 1개 소스 → 간단 브리핑(오디오 브리프/스튜디오) 제작 → 공유 링크
- 시나리오 C: 논문 PDF 2개 비교 → 공통 주장·차이점 표 → 인용 포함 요약문 초안

[프롬프트 치트시트(문서 끝에 표 형태로)]
- “요약해줘(레벨별)” / “퀴즈 만들어줘(난이도/형식)” / “블로그 초안(톤/길이)” / “회의브리핑(30초/60초)”
- “인용 달아줘(소스/페이지/문단)” / “반대 주장도 정리해줘” / “개념도/마인드맵 형태로”

[내보내기]
- 화면에 전부 못 담으면: 전체를 notebooklm_guide.md, notebooklm_guide.pdf 두 파일로 저장하고 ‘다운로드 경로’를 제공
- 화면에 담긴 경우에도 “저장용 .md”를 함께 제공

[체크리스트]
- [ ] 모든 핵심 기능 카드로 다뤘나?
- [ ] 따라하기 단계가 5클릭 이내로 끊어졌나?
- [ ] 각 기능에 예시/오류대응/고급팁이 포함됐나?
- [ ] 최종 요약·로드맵·치트시트가 포함됐나?

이 요구사항을 100% 반영해서 지금 바로 출력해줘.
```

## 🔁 단계형(분할 출력) 버전

> 길어질 때 “목차/기초 → 중급 → 고급/활용 → 치트시트/파일” 4회로 나눠 받는 프롬프트

```
[1회차] 목차 + 기초 기능만 먼저 주세요.
- 대상: 처음 사용자
- 범위: 개념, 새 노트북 만들기, 소스 추가, 기본 Q&A/요약
- 형식: 카드/체크리스트/경고/팁 포함, 클릭 경로는 ▶ 기호
- 분량: 화면 스크롤 3~4페이지 이내
- 마지막에 “다음으로 중급 기능 진행할까요?” 질문

[2회차] 중급 기능
- 인용/근거 넣기, 하이라이트/노트 정리, 검색/고급검색, 협업/공유
- 동일 형식 + 시나리오 A

[3회차] 고급/스튜디오·오디오 브리프(있다면)
- 스크립트/장면/미리보기/내보내기 + 시나리오 B

[4회차] 마무리
- 시나리오 C + 전체 요약 카드 + 1주 로드맵 + 프롬프트 치트시트
- .md/.pdf 파일도 함께 제공
```

## ✂️ 초간단(짧은) 버전

```
NotebookLM을 처음 쓰는 입문자용 전기능 가이드 만들어줘.
- 카드형/그림 설명 위주(텍스트 최소화)
- 따라하기(▶ 경로) + 체크리스트 + 흔한 실수/해결
- 기초→중급→고급 흐름, 각 단계 예시 2개
- 끝에 프롬프트 치트시트 표와 .md 파일 제공
```

## 📎 사용 팁

* 처음엔 **마스터 프롬프트**로 전체를 받고, 길다면 **단계형 버전**으로 나눠 받으세요.
* 언제든 “**.md/.pdf도 같이 주세요**”를 문장 끝에 붙이면 파일 버전을 같이 받을 확률이 높아요.
* “**클릭 경로를 ▶ 로 표기**”, “**오류/해결을 표로**”처럼 **표현 규칙을 못 박아** 주면 결과가 안정적입니다.

필요하면, 위 마스터 프롬프트에 “목차 예시”나 “카드 디자인 샘플”도 더 얹어 드릴게요.

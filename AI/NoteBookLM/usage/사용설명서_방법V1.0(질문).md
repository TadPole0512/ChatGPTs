[목표]
나는 NotebookLM을 처음 쓰는 사용자입니다. NotebookLM의 "모든 기능"을 초등학생도 이해할 수 있을 정도로 쉽고, 화면 중심(시각 효과 극대화)으로 설명해주세요. 화면 출력이 너무 길어지면 마지막에 전체 내용을 .md 및 .pdf 파일로도 만들어서 다운로드 경로를 제공해주세요.

[범위]
- 기본 개념 → 계정/접속 → 새 노트북 만들기 → 자료(문서/웹/Drive) 추가 → 소스 정리·태깅
- Q&A/브리프 생성(Notebook Guide/Source-Grounded) → 인용/근거 표시 → 노트·요약·하이라이트
- 오디오 브리프/스튜디오(있다면: 스크립트 만들기, 장면 구성, 미리보기, 내보내기)
- 협업(공유/권한) → 히스토리/버전 → 검색/고급 검색 → 설정/개인정보 → 한계/주의사항
- 고급 활용: 학습세트 만들기, 시험문제 생성, 회의 요약, 리서치 리포트/블로그 초안, 프롬프트 패턴 라이브러리

[출력 형식 (GUI 우선)]
1) 가장 위에 "📚 목차" 카드와 진행 뱃지(🔰 기초/🔥 중급/⚡ 고급)
2) 각 기능은 카드형 섹션으로:
   - 🎯 한줄 요약
   - 🛠 준비물/전제(계정, 권한, 파일 형식 등) 체크리스트
   - 🔄 따라하기(클릭 경로를 화살표로: 예) 홈 ▶ New Notebook ▶ Add sources ▶ …)
   - 👀 화면 미리보기 설명(버튼/아이콘 위치, 주의사항)
   - ✅ 완료 점검(3~5개 체크)
   - ⚠️ 흔한 실수 & 해결법(원인→해결 1줄)
   - 💡 응용/고급 팁(프롬프트 예시 2~3개)
3) 표는 3~6열, 10행 이내(필요 시 표를 나눠서)
4) 중요 경고는 붉은 경고 상자(⚠️), 참고는 파란 정보 상자(💡)
5) 단계가 긴 파트는 "🔍 자세히 보기" 접기/펼치기(아코디언)로 나눠서 출력
6) 마지막에 "전체 요약 카드" + "학습 로드맵(1주 코스)" + "프롬프트 치트시트" 포함

[톤 & 난이도]
- 초등학생도 이해: 어려운 말 금지, 쉬운 비유("노트북=폴더, 소스=책/자료") 사용
- 한국어 기본, 영어 메뉴명은 괄호 병기 (예: 소스 추가(Add sources))
- 단계는 번호로, 클릭/입력은 **굵게** 표시

[검증/근거]
- 각 기능 설명에 실제 버튼/메뉴 경로를 굵게 인라인 표기
- 기능 사용 예시는 2가지 이상(학습/업무)
- 중요한 한계(저작권, 개인정보, 환각 가능성)와 회피법 명시

[샘플 시나리오 (필수 포함)]

- 시나리오 B: 회의록 2개 + 웹페이지 1개 소스 → 간단 브리핑(오디오 브리프/스튜디오) 제작 → 공유 링크
- 시나리오 C: 논문 PDF 2개 비교 → 공통 주장·차이점 표 → 인용 포함 요약문 초안

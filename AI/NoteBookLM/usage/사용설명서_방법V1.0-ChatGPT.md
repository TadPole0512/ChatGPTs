## 1. 계정 만들기 및 접속 (🔰 기초)

**🎯 한줄 요약**: Google 계정만 있으면 바로 NotebookLM에 로그인해 시작할 수 있어요.

**🛠 준비물/전제 체크리스트**
- [ ] Google 계정 (한국 계정 OK)
- [ ] PC 크롬 브라우저(권장)
- [ ] 인터넷 연결

**🔄 따라하기 (클릭 경로)**
**홈페이지** ▶ **NotebookLM 접속** ▶ **Google 로그인** ▶ **시작하기**

**👀 화면 미리보기 설명**
- 우상단에 **프로필 아이콘**, 중앙에 **“새 노트북 만들기”** 버튼이 보여요.
- 한국어 UI로 안내 문구가 나옵니다.

**✅ 완료 점검**
- [ ] 로그인 완료(우상단 계정 표시)
- [ ] 새 노트북 만들기 버튼 확인
- [ ] 필요 시 권한 동의 완료

**⚠️ 흔한 실수 & 해결법**
- 원인: 업무/학교 계정에서 차단됨 → 해결: 개인 계정 사용 또는 관리자에 허용 요청
- 원인: 빈 화면 → 해결: 브라우저 업데이트/새로고침

**💡 응용/고급 팁**
- 여러 계정 전환은 **프로필 아이콘**에서 가능
- 개인 정보가 민감하면 개인정보 포함 자료 업로드는 지양

## 2. 새 노트북 생성 (🔰 기초)

**🎯 한줄 요약**: 주제별로 노트북을 만들어 자료와 Q&A를 모아두세요.

**🛠 준비물**
- 노트북 주제 아이디어 (예: 과학 숙제)

**🔄 따라하기**
**홈** ▶ **새 노트북** ▶ **제목 입력** ▶ (원하면 즉시 자료 추가)

**👀 화면 미리보기 설명**
- 좌측: **소스(Sources)**, 중앙: **채팅/답변**, 우측: **스튜디오(Notes/Overview)**

**✅ 완료 점검**
- [ ] 홈 목록에 새 노트북 표시
- [ ] 제목을 의미 있게 변경

**⚠️ 흔한 실수 & 해결법**
- “제목 없음”이 많음 → 각 노트북 열고 상단 제목을 바꿔 정리

**💡 응용/고급 팁**
- 과목/프로젝트별로 여러 노트북 생성해 분류

## 3. 자료 추가 (문서·웹·Google Drive) (🔰 기초)

**🎯 한줄 요약**: PDF/텍스트/웹 URL/Drive 문서를 **소스로 추가**하면 AI가 이해하고 답변해요.

**🛠 준비물**
- 파일(PDF/TXT/MD) 또는 **웹 URL**, **Google Drive** 접근 권한

**🔄 따라하기**
노트북 좌측 **소스 패널** ▶ **➕ 소스 추가(Add source)** ▶ **파일/URL/Drive 선택** ▶ **가져오기**

**👀 화면 미리보기 설명**
- 소스 목록에 **파일명/아이콘**이 보이고, 클릭하면 미리보기/요약 표시

**✅ 완료 점검**
- [ ] 소스 목록에 정상 등록
- [ ] 미리보기/요약 확인
- [ ] (Drive) 원본 수정 시 **동기화** 버튼 확인

**⚠️ 흔한 실수 & 해결법**
- 스캔 PDF(이미지) → OCR된 텍스트 PDF 사용 권장
- 로그인 필요한 웹페이지 → 공개 페이지만 추가 가능
- YouTube → 자막 기반으로만 텍스트 반영

**💡 응용/고급 팁**
- 여러 URL을 줄바꿈으로 **한 번에 추가**
- 문서가 크면 **여러 파일로 분할** 후 업로드

## 4. 노트북 정리 (태그 & 구성) (🔰 기초)

**🎯 한줄 요약**: 태그 기능이 없어도 **이름 규칙+검색**으로 깔끔히 관리해요.

**🛠 준비물**
- 정리할 노트북 목록과 이름 규칙(예: `[과목] 주제`)

**🔄 따라하기**
**홈** ▶ **내 노트북** ▶ **정렬/검색** ▶ **제목 규칙 적용**

**👀 화면 미리보기 설명**
- 카드에 제목, 소스/노트 개수, 공유 상태(🔒/👥/🌐) 표시

**✅ 완료 점검**
- [ ] 제목 중복/“제목 없음” 제거
- [ ] 정렬 기준(제목/최근) 설정
- [ ] 공유 상태 재확인

**⚠️ 흔한 실수 & 해결법**
- 정리 전 공유 → 공유 전 제목/불필요 노트 정리

**💡 응용/고급 팁**
- 중요 노트 제목 앞에 `1.`, `★` 등 접두사로 상단 배치

## 5. 노트북 Q&A와 질문하기 (🔥 중급)

**🎯 한줄 요약**: 자료 기반으로 **자연어 질문**하면 **출처 달린 답변**을 받아요.

**🛠 준비물**
- 소스가 추가된 노트북, 질문거리

**🔄 따라하기**
**채팅 입력창** ▶ 질문 입력 ▶ **Enter** ▶ 답변 **인용 번호** 클릭해 출처 확인 ▶ 후속 질문

**👀 화면 미리보기 설명**
- 대화 말풍선에 **¹²³** 같은 인용 번호, 클릭 시 원문 하이라이트

**✅ 완료 점검**
- [ ] 답변 수신
- [ ] 인용 번호로 출처 확인
- [ ] 후속 질문으로 심화

**⚠️ 흔한 실수 & 해결법**
- 질문이 막연함 → 주제/범위/형식 지정해 구체화

**💡 응용/고급 팁 (프롬프트 예시)**
- “이 문서에서 **환경오염**만 요약해줘”
- “A와 B의 **차이점**을 표로 정리해줘”

## 6. AI 요약 생성 (🔥 중급)

**🎯 한줄 요약**: 긴 문서도 **한눈에 요약**.

**🛠 준비물**

**🔄 따라하기**
**채팅** ▶ “요약해줘(길이/형식 지정)” ▶ 결과 검토 ▶ **짧게/길게** 재요청

**👀 화면 미리보기 설명**
- 단락/불릿 요약 + 인용 번호

**✅ 완료 점검**
- [ ] 핵심 포함 여부
- [ ] 출처 일치 확인
- [ ] 길이/형식 적합

**⚠️ 흔한 실수 & 해결법**
- 모호한 요약 → “핵심 3가지” 등 **개수 지정**

**💡 응용/고급 팁**
- “각 **챕터**를 한 줄씩”
- “**학생용** 쉬운 말로”
- “**수치**도 포함해 요약”

## 10. 노트 작성 및 정리 (🔥 중급)

**🎯 한줄 요약**: **직접 노트**를 쓰거나 **AI 답변을 노트로 저장**.

**🔄 따라하기 – 새 노트 작성**
우측 **Studio › Notes** ▶ **+ 노트 추가** ▶ 제목/내용 입력(자동 저장)

**🔄 따라하기 – AI 답변 저장**
좋은 답변 ▶ **노트로 저장(💾)** ▶ 목록에서 확인 (편집 불가 노트)

**✅ 완료 점검**
- [ ] 새 노트 저장/표시
- [ ] 저장된 답변 노트 확인

**⚠️ 흔한 실수 & 해결법**
- 저장 안 된 줄 앎 → 자동 저장, 잠시 후 “저장됨” 표시 확인

**💡 응용/고급 팁**
- 여러 노트를 **하나로 합치기**, **개요 만들기** 등 퀵액션
- 노트를 **소스로 변환**해 재활용

## ⚠️ 한계와 문제 해결법 (⚡ 고급)

> **환각(자료에 없는 내용)** → **인용 클릭**으로 원문 대조, “출처 없는 내용 빼고 다시” 재요청  
> **정보 누락** → “~~도 포함해줘” 재질문, 마인드맵/목차로 구조 점검  
> **저작권** → 공유/공개 시 **출처 표시**, 원문 **직접 인용 최소화**  
> **개인정보/보안** → 공유 범위(🔒/👥/🌐) 상시 점검, 민감 자료는 비공개 유지

**✅ 완료 점검**

## 시나리오 A: 과학 챕터 요약 & 퀴즈 생성

**목표**: 과학 교과서 챕터 3개 업로드 → 요약 노트 → **퀴즈 5문항** 자동 생성

**절차**
1) **새 노트북** 생성, 챕터3개 **PDF 업로드**  
2) “각 챕터를 **한 줄 요약**” 요청 → **노트 저장**  
3) “요약 기반 **퀴즈 5문항**(객관식 4지선다)” 요청  
4) 퀴즈를 노트로 저장 또는 복사해 배포

**응용 프롬프트**
- “오답 선택지도 **헷갈리게** 만들어줘”
- “정답 근거에 **인용 번호** 달아줘”

## 시나리오 B: 회의록+기사 브리핑 & 오디오 공유

**목표**: 회의록 2개 + 웹기사 1개 → **브리핑 문서** → **오디오 브리프** 생성 → 공유 링크

**절차**
1) 소스 3개 추가(회의록2·URL1)  
2) **Briefing 생성** → 필요한 문장 편집  
3) **Audio Overview 생성**(Brief 모드) → MP3 다운로드/링크 공유  
4) 노트북 **Viewer 공유**로 팀에 배포

**응용 프롬프트**
- “경영진에게 **1분 요약**으로 설명”
- “리스크/액션아이템 **표**로 추가”

---

## 시나리오 C: 논문 2개 비교 · 인용 요약

**목표**: PDF 2개 비교 → **공통 주장/차이점 표** → **인용 포함 요약문**

**절차**
1) 논문 A/B 업로드  
2) “A와 B의 **공통점/차이점**을 **표(3열)**로” 요청  
3) “표를 바탕으로 **요약문** 작성, **인용 표시** 유지”  
4) 요약문을 노트로 저장

**응용 프롬프트**
- “두 연구의 **한계/추가 연구** 제시”
- “**중요 수치**만 모아 표로”

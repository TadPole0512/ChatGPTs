## 🧠 7. (고급) 창의 아이디어 강제 생성 모듈

> 아래 중 **최적 2개 공식을 선택**해 혼합 적용하고, **아이디어 10개 이상** 산출(필요 시 접기/펼치기)

* **통찰 공식(GI)**: 관찰(O)×연결(C)×패턴(P)×종합(S) / (가정(A)+편향(B))
  · 적용: 관찰→연결→패턴→종합 순, 마지막에 “가정/편향 제거” 로그 출력
* **역발상 공식(PR)**: 금기/당연/관습을 뒤집어 5분류 아이디어 도출
* **조합 공식(CC)**: 기능×산업×사용자군 교차표로 3×3 이상 매트릭스 발상
* **시나리오 변형(IS)**: 초보/중수/고수·개인/팀·모바일/데스크톱별 재구성

**출력 지시**

* 아이디어는 **카드 10개+한줄 근거**로, 상·중·하 임팩트 태그와 적용 난이도 표시
* 문서 본문과 직접 연결되는 **실행 예시**(버튼/메뉴/자동화 규칙)까지 제시

## 1) 핵심형(빠르게 끝내기)

```
너는 초보자도 이해하는 시각화 중심 문서 디자이너야.
[대상/주제]: "<여기에 제품/기능/작업명>"
요구:
- 요약 대시보드(목적/난이도/소요시간/선행조건/결과물)
- 작업 흐름(시작→설정→실행→검증→산출물) 다이어그램+카드
- 각 단계: ▶클릭 경로/버튼명, 스크린샷 지점, 체크박스, 경고/팁 박스
- 문제해결 표(증상/원인/조치) + 빠른점검 체크리스트
- 길면 화면 우선 출력, 끝에 전체 .md 본문 제공
톤: 초등학생도 이해, 불필요한 말 생략
출력: 카드/박스/표 위주로 바로 보여줘.
```

## 2) 워크북형(따라하며 학습)

```
너는 코치처럼 단계별 실습을 이끄는 튜터야.
[대상/주제]: "<여기에 제품/기능/작업명>" 실습 워크북
구성:
1) 준비물 카드(계정/권한/데이터/환경 체크박스)
2) 단계별 실습: 목표→이유→절차(▶클릭 경로)→검증 체크
3) 미니 퀴즈 3문항 + 과제(제출 기준 포함)
4) 흔한 오류 5개(증상/원인/즉시조치/근본해결)
5) 진행률·배지 표기
출력: 화면 우선, 끝에 .md 전체본 동봉.
```

## 3) 레퍼런스형(완전 참고서)

```
너는 기술 문서 작성가야.
[대상/주제]: "<여기에 제품/기능/작업명>" 레퍼런스
목차:
개요/용어집/역할·권한/설치·접속/기능별 상세(표: 필드·옵션·기본값·권장값·주의)/
고급 설정/자동화/FAQ/문제해결/부록(.md)
규칙:
- 모든 기능은 표로 정리 + 경고/팁 박스
- 흐름도/시퀀스 다이어그램 포함
출력: 길면 마지막에 .md 전체 제공.
```

## 4) 고급 아이디어 강화(옵션: 창의 개선안 뽑기)

```
아래 2개 공식을 혼합해 [대상/주제] 개선 아이디어 10개 이상을 카드로 제시:
- 통찰(GI): 관찰→연결→패턴→종합, 마지막에 가정/편향 제거 로그
- 조합(CC): 기능×사용자군×산업 3×3 이상 교차표로 발상
각 카드: 한줄 근거, 적용 난이도, 예상 임팩트(상/중/하), 바로 적용 예시(버튼/메뉴/자동화 규칙).
```

## 5) 검증·품질 점검만 따로 요청

```
아래 문서 초안의 품질을 '검증 체크리스트'로 점검하고 수정안을 제시해줘.
요구:
- 누락/모호/가정/출처불명 항목 표시(⚠️)

- 초보자 난이도/가독성/시각화(카드·박스·다이어그램) 점수(10점 만점)와 개선 팁 5개
- 수정본을 반영한 .md 최종본 재생성
초안:
<<<여기에 초안 붙여넣기>>>
```

## 6) “NotebookLM 스튜디오 보고서 생성” 즉시 실행 예시

```
[대상/주제]: "NotebookLM 스튜디오에서 보고서 생성하는 방법"
형식: 1) 요약 대시보드 2) 작업 흐름 다이어그램 3) 단계별 카드(▶클릭 경로/스크린샷 지점/체크박스)
4) 실습 미션(입력/기대결과/검증법) 5) 문제해결 표 6) 길면 .md 전체본 첨부.
```

### 쓰는 요령 (초간단)

* 대괄호만 바꿔서 복붙 → 길어지면 .md 동봉 지시를 항상 넣기
* 클릭 경로/버튼명/스크린샷 지점은 반드시 요구하기
* 결과 받으면 ⑤·⑦ 프롬프트로 **검증 → 리팩터링** 순서로 바로 다듬기

필요하면 위 7개를 당신 프로젝트에 맞게 변수화한 “한 장짜리 .md 프롬프트팩”으로 묶어 드릴게.

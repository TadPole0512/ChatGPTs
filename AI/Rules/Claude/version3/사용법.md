# 📱 앱 사용법·서비스 매뉴얼 전문 제작 지침 v3.0

## 🎯 핵심 미션
**실무 즉시 적용 가능한, 검증된 최신 정보 기반의 완전한 사용 매뉴얼 제작**  
→ 이론 설명 최소화, 실행·검증·트러블슈팅 중심 구성

## 📐 필수 준수 사항

### 1️⃣ 최신성 & 검증 원칙
- **2025년 자료 최우선** (공식 문서·릴리즈 노트·최신 버전 스펙)
- 버전·스펙·기능·UI는 **웹 검색 필수** → 출처 URL + 확인 일시(YYYY-MM-DD HH:mm KST)
- 교차 검증: 공식 문서 + 실제 테스트 가능 예시 + 커뮤니티 최신 피드백(3개월 이내)
- 불확실한 정보는 **[2025년 확인 필요]** 배지 표기

### 2️⃣ 문서 구조 템플릿 (25,000자+ 기준)

```markdown
# [앱명/서비스명] 완전 사용 매뉴얼 (2025년 최신판)

## 📑 목차 (TOC)
- 계층형 3단계, 각 항목에 예상 소요 시간 표기

## 🎯 문서 개요
- 대상 독자 (초급/중급/고급 구분)
- 전체 학습 소요 시간
- 사전 준비 사항 체크리스트

## 🚀 빠른 시작 (Quick Start)
- 5분 내 핵심 기능 체험 시나리오
- Before → Action → Verify 패턴

## 📱 Tier 1: 기본 사용법 (난이도 ★☆☆)
### 1.1 설치·초기 설정
- OS별 설치 경로 (Windows/Mac/Linux/Android/iOS)
- 시스템 요구사항 표 (CPU/RAM/저장공간/네트워크)
- 초기 설정 체크리스트 (10단계 이내)

### 1.2 인터페이스 이해
- GUI 주요 영역 스크린샷 (번호 라벨링)
- 메뉴·버튼 기능 표 (이름-기능-단축키)

### 1.3 핵심 기능 3가지
- 기능별 사용 시나리오 (목적-절차-결과 검증)

## 🔧 Tier 2: 실무 활용법 (난이도 ★★☆)
### 2.1 통합·연동 설정
- 외부 서비스 연결 (API키·OAuth·플러그인)
- 단계별 스크린샷 + 오류 대응

### 2.2 생산성 향상 팁
- 시간 절약 워크플로우 (Before/After 비교)
- 단축키 치트시트 표

### 2.3 실전 시나리오 5개
- 업무별 구체적 예시 (코드/문서/데이터 작업)

## ⚙️ Tier 3: 고급 최적화 (난이도 ★★★)
### 3.1 성능 튜닝
- 설정 파일 수정 (경로·파라미터·주석 포함)
- 측정 가능한 성능 지표 (처리 시간·메모리 사용량)

### 3.2 자동화·스크립팅
- 전체 실행 가능 코드 (Python/Shell/PowerShell)
- 입력 예시 2개 + 출력 + 에러 케이스

### 3.3 엔터프라이즈 기능
- 팀 협업 설정·권한 관리·감사 로그
```

### 3️⃣ 콘텐츠 작성 규칙

#### 📌 Before → Action → Verify 패턴 적용
```markdown
**Before**: 현재 상태 또는 준비물
**Action**: 
1. [GUI 메뉴 경로] 또는 `명령어`
2. [입력값] 예시 포함
3. [확인 버튼] 클릭

**Verify**: 
- 화면에 "Success" 메시지 출력
- `명령어`로 상태 확인 → 예상 결과
```

#### 📌 코드·스크립트 제공 기준
- **전체 소스 필수** (부분 코드 절대 금지)
- 파일 경로·실행 환경·의존성 명시
```python
# 파일: /home/user/scripts/automation.py
# 환경: Python 3.11+, requests 라이브러리 필요
# 실행: python automation.py --input data.json

import requests
import json

def main():
    # 전체 실행 가능 코드 (50~200줄)
    pass

if __name__ == "__main__":
    main()

# 샘플 입력 1: {"key": "value"}
# 예상 출력: {"status": "success", "data": [...]}
# 에러 케이스: {"error": "Invalid API key"} → API 키 재확인
```

#### 📌 시각화 필수 요소
- **표**: 3~6열, 제브라 스타일, 헤더 고정
- **카드**: 기능별·난이도별 그룹화
- **배지**: `[최신]` `[필수]` `[고급]` `[실험]` `[2025년 확인 필요]`
- **상태 표시**: ✅ 완료 / ⏳ 진행 중 / ⚠️ 주의 / ❌ 오류
- **프로그레스 바**: 학습 진도·설치 단계

#### 📌 스크린샷·다이어그램 대체 표현
실제 이미지 첨부가 불가능할 경우:
```markdown
📸 [스크린샷 위치: 메인 화면 - 상단 메뉴 바]
- ① File 메뉴 (좌측 상단)
- ② Edit 버튼 (툴바 중앙)
- ③ 상태 패널 (우측 하단, 초록색 "Ready" 표시)
```

### 4️⃣ 난이도 체계 (Tier System)

| Tier | 난이도 | 대상 | 핵심 내용 | 비율 |
|------|--------|------|-----------|------|
| **Tier 1** | ★☆☆ | 초급 (신규 사용자) | 설치·UI·핵심 기능 3가지 | 30% |
| **Tier 2** | ★★☆ | 중급 (3개월+ 경험) | 통합·자동화·실무 시나리오 | 50% |
| **Tier 3** | ★★★ | 고급 (1년+ 파워유저) | 성능 튜닝·스크립팅·엔터프라이즈 | 20% |

### 5️⃣ 품질 검증 3단계

#### ① 기본 검증 (공식 정보)
- 공식 문서 URL 3개 이상 확인
- 최신 버전 릴리즈 노트 교차 검증
- 메뉴·UI 용어 정확성 (스크린샷 또는 실제 실행 확인)

#### ② 실험 검증 (재현 가능성)
- 제시한 명령어·설정이 실제 동작하는지 확인
- 샘플 데이터로 입출력 테스트
- 에러 케이스 재현 → 해결 방법 검증

#### ③ 커뮤니티 검증 (실사용자 피드백)
- Stack Overflow·Reddit·공식 포럼 최신 이슈 (3개월 이내)
- 표본: 관련 질문 10건 이상, 답변 품질 ★3.5+ 기준
- 버전별 호환성 주의사항 반영

### 6️⃣ 안전·제약 사항 의무 표기

```markdown
## ⚠️ 주의사항 & 제약

### 🚫 금지 사항

- 프로덕션 환경 직접 테스트 금지
- 라이선스 위반 설정 예시

### 🔒 보안 고려사항
- API 키·비밀번호 하드코딩 금지
- 환경변수 사용 권장: `export API_KEY=xxx`
- SSL/TLS 인증서 검증 활성화

### 🛟 응급 대응
- 서비스 중단 시: `systemctl restart service-name`

### 📋 보증 유의
- 무료 플랜 제약: 월 요청 1,000건, 저장 용량 5GB
- 유료 전환 시 마이그레이션 가이드
```

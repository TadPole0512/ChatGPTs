📖 Chapter 1: Claude Skills란 무엇인가?

1-1. Claude Skills의 정의

Claude Skills를 가장 쉽게 설명하면 다음과 같습니다:

Claude에게 주는 '업무 매뉴얼' 또는 '전문 지식 패키지'

실제 업무 상황을 생각해보세요. 신입사원을 채용했을 때 어떻게 교육하나요?

  1. 말로 설명한다
  2. 매뉴얼을 준다
  3. 과거 사례를 보여준다
  4. 작업 체크리스트를 제공한다
  5. 참고 자료를 준비한다

Claude Skills는 정확히 이 모든 과정을 AI에게 적용하는 것입니다. Claude를 여러분의 업무 방식에 맞춰진 '전문가'로 만드는 것이죠.

1-2. 왜 Claude Skills가 필요한가?

만약 Claude Skills 없이 매번 같은 작업을 한다면:

스킬 없이 사용할 때
사용자: "우리 회사 스타일로 보고서 써줘"
Claude: 일반적인 형식의 보고서 생성

다음 날
사용자: "우리 회사 스타일로 또 다른 보고서 써줘"
Claude: 또 다른 형식으로 생성 (이전과 다를 수 있음)

또 다음 날
사용자: "같은 형식으로..."
Claude: "제가 지난 형식을 기억하지 못합니다..."

Claude Skills를 사용하면:

스킬 사용할 때
처음: Skills 설정 (1회만)
사용자: "우리 회사 스타일로 보고서 써줘"
Claude: Skills를 자동으로 로드 → 정확한 형식의 보고서 생성

다음 날
사용자: "보고서 만들어줘"
Claude: 같은 Skills 자동 로드 → 동일한 형식의 보고서 생성

📌 계속 같은 일관성 유지!

1-3. Claude Skills의 핵심 특징

특징 1: 자동 적용

스킬을 설정하면 관련된 요청을 할 때 Claude가 자동으로 적용합니다. "이 스킬을 써줘"라고 말할 필요가 없습니다.

특징 2: 여러 플랫폼 호환성

  • Claude Desktop (윈도우/맥)
  • Claude.ai 웹사이트
  • Claude API
  • Claude Code
  • 모든 플랫폼에서 동일하게 작동합니다

특징 3: 재사용 가능

한 번 만든 스킬을 팀원들과 공유 가능하며, 다른 프로젝트에도 활용할 수 있습니다.

특징 4: 확장 가능

텍스트 지침만으로도 기본적으로 작동하며, 점진적으로 데이터, 코드, 예시를 추가하면서 기능을 확장할 수 있습니다.

📊 Chapter 2: Claude Skills로 할 수 있는 일

2-1. 기본적인 활용 분야

🎨 마케팅 분야

  • 상품 소개 문구 생성: 회사의 톤앤매너와 스타일 규칙을 적용하여 일관된 마케팅 카피 생성
  • A/B 테스트 카피: 여러 버전의 광고 문구를 자동으로 생성
  • 이메일 템플릿: 고객별 맞춤 이메일의 기본 틀 자동 생성
  • 소셜 미디어 콘텐츠: 플랫폼별 특성에 맞는 포스트 작성

📄 문서 작성

  • 보고서 자동화: 주간/월간 보고서를 정해진 형식대로 생성
  • 회의록 정리: 음성/텍스트를 회사 표준 회의록으로 변환
  • 이력서 검수: 이력서를 표준 형식으로 변환하고 문법 검사
  • 제안서 작성: 고객별 맞춤 제안서 초안 자동 생성

💻 개발/기술

  • API 문서 생성: 코드에서 자동으로 API 문서 초안 작성
  • 코드 리뷰: 팀의 코드 리뷰 기준에 따라 자동 검토
  • 기술 문서 작성: 일관된 포맷의 기술 문서 생성
  • 버그 리포트 정리: 난잡한 버그 리포트를 표준 형식으로 변환

💼 영업/고객관리

  • 제안 이메일: 고객별로 맞춤화된 제안 이메일 자동 생성
  • 고객 응대 문구: 다양한 고객 요청에 대한 표준 응대 문구
  • 계약서 검토: 회사 계약 정책에 따른 자동 검토
  • CRM 데이터 정리: 고객 정보를 표준 형식으로 입력

👥 인사/HR

  • 면접 질문 생성: 공고와 지원자 정보에 맞춘 맞춤 면접 질문
  • 채용공고 작성: 일관된 형식의 채용공고 자동 생성
  • 온보딩 자료: 신입사원 교육용 자료 자동 생성
  • 직원 평가: 평가 기준에 따른 자동 평가안 작성

🔧 Chapter 3: 기본 개념 이해하기

3-1. Claude Skills의 구조

Claude Skills는 폴더 구조를 기반으로 합니다. 가장 단순한 형태부터 복잡한 형태까지 다음과 같이 구성됩니다.

최소 구조 (가장 간단함)

my-skill/ └── SKILL.md

이것만으로도 기본적인 스킬이 작동합니다.

기본 구조

my-skill/ ├── SKILL.md # 필수: 스킬의 모든 지침 └── REFERENCE.md # 선택: 추가 참고 자료

완전한 구조

my-skill/ ├── SKILL.md # 필수: 스킬의 정의와 지침 ├── REFERENCE.md # 선택: 상세한 참고 자료 ├── examples/ # 선택: 입출력 예시 │ ├── good_input.txt │ ├── bad_input.txt │ ├── good_output.txt │ └── bad_output.txt ├── scripts/ # 선택: 실행 가능한 코드 │ ├── process.py │ ├── analyze.py │ └── requirements.txt └── resources/ # 선택: 이미지, 템플릿, 설정 ├── logo.png ├── template.docx └── config.json

3-2. 각 파일의 역할

파일 역할 중요도 필수 여부
SKILL.md 스킬의 정의와 지침 ⭐⭐⭐⭐⭐ 필수
REFERENCE.md 추가 참고 자료 ⭐⭐⭐ 선택
examples/ 입출력 예시 ⭐⭐⭐⭐ 선택
scripts/ 실행 가능한 코드 ⭐⭐⭐⭐⭐ 선택
resources/ 보조 자료 ⭐⭐⭐ 선택
💡 핵심 포인트: SKILL.md는 필수이며, 나머지 파일들은 필요에 따라 선택적으로 추가할 수 있습니다. 하지만 가능한 많이 추가할수록 Claude의 이해도가 높아집니다.

3-3. Claude가 Skills를 사용하는 과정

단계 1: 메타데이터 읽기 (항상)

사용자의 요청이 들어오면, Claude는 모든 활성화된 스킬의 name과 description을 읽습니다. "이 요청에 맞는 스킬이 있나?" 판단합니다. 이것이 매우 빨리 진행되어 사용자는 지연을 느끼지 못합니다.

단계 2: 스킬 선택

가장 관련성 높은 스킬을 선택합니다. 여러 스킬이 관련되면 가장 적합한 것부터 로드합니다.

단계 3: 스킬 파일 로드 (선택적)

선택된 스킬의 SKILL.md 파일을 읽습니다. 해당 작업에 필요한 지침을 모두 이해합니다.

단계 4: 추가 정보 로드 (필요시)

더 상세한 정보가 필요하면 REFERENCE.md를 읽습니다. 예시를 확인하려면 examples/ 파일을 읽습니다. 코드를 실행해야 하면 scripts/ 파일을 로드합니다.

단계 5: 작업 실행

스킬의 지침을 따라 사용자의 요청을 처리합니다. 필요하면 scripts/의 코드를 실행합니다.

단계 6: 결과 제공

완성된 결과물을 사용자에게 제공합니다. 파일이 생성되었으면 그 파일의 경로를 알려줍니다.

3-4. Description의 중요성

스킬의 가장 중요한 부분은 description 필드입니다.

Claude가 100개 이상의 스킬 중에서 여러분의 요청에 맞는 스킬을 찾아내는 것은 이 description을 읽고 판단하기 때문입니다.

description: 문서를 만듭니다.
description: 팀의 주간 업무 현황을 '진행 중', '완료', '이슈' 세 가지 항목으로 구분하여 정해진 형식의 마크다운 보고서를 작성합니다.

📁 Chapter 4: 폴더와 파일 구성하기

4-1. 작업 환경 준비하기

필요한 도구

  1. 텍스트 에디터 (파일을 만들고 편집하기 위해)
    • 추천: VS Code (무료, 강력함) https://code.visualstudio.com/
    • 또는: 메모장 (윈도우 기본 제공)
    • 또는: TextEdit (맥 기본 제공)
  2. 압축 해제 도구
    • 윈도우: 기본 제공 (마우스 우클릭 > 압축 해제)
    • 맥: 기본 제공 (압축 파일을 더블클릭)
  3. Claude Desktop 또는 웹사이트
    • Claude Desktop: https://claude.ai/download
    • 웹사이트: https://claude.ai

4-2. 스킬 폴더 위치 이해하기

Claude가 스킬을 찾는 위치는 정해져 있습니다. 반드시 이 위치에 폴더를 만들어야 합니다.

Windows 사용자

C:\Users\[사용자명]\.claude\skills\

예: C:\Users\john\.claude\skills\

Mac 사용자

/Users/[사용자명]/.claude/skills/

예: /Users/john/.claude/skills/

⚠️ 숨김 폴더 주의: .claude 폴더는 숨김 폴더입니다. 처음에는 보이지 않을 수 있습니다. 아래 방법을 따라 찾으세요.

❓ Chapter 10: 자주 묻는 질문 (FAQ)

Q1: 스킬과 프롬프트의 차이가 뭔가요?

프롬프트 스킬
입력 방식 매번 입력 한 번 설정
일관성 낮음 완벽함
효율성 낮음 높음
관리 어려움 쉬움

Q2: 스킬은 어디에 저장되나요?

Windows: C:\Users\[사용자명]\.claude\skills\

Mac: /Users/[사용자명]/.claude/skills/ 또는 ~/.claude/skills/

Q3: 몇 개까지 스킬을 만들 수 있나요?

제한이 없습니다! 원하는 만큼 만들 수 있습니다. 하지만 권장사항은 다음과 같습니다:

  • 가장 자주 사용하는 스킬 10-20개 활성화
  • 나머지는 필요할 때만 활성화
  • 정기적으로 사용하지 않는 스킬은 비활성화

Q4: 팀원들과 스킬을 공유할 수 있나요?

물론입니다! 여러 방법이 있습니다:

  1. ZIP 파일로 공유: 스킬 폴더를 ZIP 압축 후 이메일로 전송
  2. GitHub로 공유: GitHub에 업로드하여 링크 공유
  3. 문서로 공유: SKILL.md 내용을 문서로 변환

Q5: 스킬이 오작동하면?

  1. 스킬만 활성화하고 테스트
  2. Claude의 답변 아래 "Thinking" 섹션 확인
  3. Description 수정 (더 구체적으로)
  4. 메타데이터 형식 재확인

Q6: 스킬에서 외부 API를 호출할 수 있나요?

가능합니다! Python의 requests 라이브러리를 사용하여 API 호출을 할 수 있습니다. 다만 API 키는 절대 SKILL.md에 적지 말고 환경 변수로 관리해야 합니다.

Q7: 스킬을 비활성화해도 파일이 삭제되나요?

아니요, 완전히 안전합니다! 비활성화는 사용하지 않기만 하는 것이고, 파일은 그대로 보관됩니다. 필요하면 언제든 다시 활성화할 수 있습니다.

Q8: 스킬은 최신 Claude 모델에서만 작동하나요?

모든 Claude 모델에서 작동합니다 (Claude 3.5 Sonnet, Claude 3 Opus/Sonnet/Haiku 등). 더 강력한 모델일수록 복잡한 스킬도 잘 처리합니다.

📌 마지막 팁: 완벽한 스킬 만들기 5원칙

원칙 1: "한 가지만!" (Single Responsibility)

한 스킬 = 한 가지 명확한 목적

원칙 2: "설명이 전부!" (Description is Everything)

Description이 좋으면 스킬도 좋다

원칙 3: "보여주기가 최고!" (Show, Don't Tell)

텍스트 지침 10줄보다 좋은 예시 1개

원칙 4: "작게 시작!" (Start Small)

완벽한 스킬보다 작동하는 스킬부터

원칙 5: "계속 개선!" (Iterate)

테스트 → 피드백 → 수정의 반복

# Claude Skills 완벽 초급자 가이드 - 따라하기 + 심화

## 프롤로그: 이 가이드가 당신을 위한 이유

"Claude Skills"라는 말을 들었을 때, 혹시 이런 생각이 들었나요?

- "뭔가 복잡할 것 같은데...?"
- "기술적인 것들이 많이 필요한 건 아닐까?"
- "내가 할 수 있을까?"

**걱정하지 마세요.** 이 가이드는 그런 걱정을 완전히 없애주기 위해 만들어졌습니다.

마치 요리 레시피처럼, 우리는 한 단계씩, 아주 자세하게 함께 나아갈 것입니다. 이 문서를 따라하면, **아무런 기술 배경 없어도** 당신의 첫 번째 Claude Skill을 만들 수 있습니다.

# PART 1: 초급자를 위한 기초 개념

## Chapter 1: Claude Skills 개념 완전 정리

### 1.1 Claude Skills란 정확히 뭔가요?

매일 반복되는 작업을 생각해보세요.

예를 들어, **매주 월요일 오전에 지난주 업무 현황을 보고서로 정리**하는 일이 있다고 합시다.

**전통적인 방식:**
```
월요일 09:00
"지난주 일들을 정리해야 하는데... 
어떤 형식으로 정리했더라?"
↓
지난주 보고서 찾아보기
↓
"아, 이 형식이었지... 다시 만들어야겠네"
↓
30분 소요
```

**Claude Skills를 사용한 방식:**
```
월요일 09:00
"Claude, 지난주 업무 보고서 만들어줘"
↓
Claude: "주간 업무 보고서 스킬 활성화!"
↓
2초 만에 완성!
```

**Claude Skills는 이렇습니다:**

> Claude에게 "이건 이런 형식으로, 이런 내용으로, 이런 방식으로 해줘" 하는 **일종의 업무 매뉴얼**입니다.
> 한 번 만들면 계속 사용할 수 있고, Claude가 자동으로 알아서 적용합니다.

### 1.2 Claude Skills가 일반 프롬프트와 다른 점

**시나리오를 통한 비교:**

#### 일반 프롬프트 방식:
```
사용자: "이 데이터를 분석해서 보고서 만들어줄래?
- 형식: Word 문서
- 색상: 회사 브랜드 컬러(파란색)
- 제목 폰트: Arial Bold 16pt
- 본문 폰트: Arial 11pt
- 섹션별로 구분
- 마지막에 결론과 제안사항 포함
- 회사 로고는 우측 하단에"

Claude: "네, 만들어드릴게요!"

다음날:
사용자: "또 다른 데이터도 분석해서 보고서 만들어줄래?
- 형식: Word 문서
- 색상: 회사 브랜드 컬러(파란색)
...

[매번 동일한 설명을 반복해야 함]"
```

#### Claude Skills 방식:
```
[처음 한 번만] 
스킬 제작: "우리 회사 스타일 보고서 작성 스킬" 만들기
- 형식, 색상, 폰트, 로고 배치 모두 정의

[이제부터 계속]
사용자: "데이터 1을 분석해서 보고서 만들어줘"
Claude: [스킬 자동 적용] → 완성!

사용자: "데이터 2를 분석해서 보고서 만들어줘"
Claude: [스킬 자동 적용] → 완성!

사용자: "데이터 3을 분석해서 보고서 만들어줘"
Claude: [스킬 자동 적용] → 완성!

[매번 동일한 형식으로 자동 적용됨]
```

**핵심 차이:**

| 항목 | 일반 프롬프트 | Claude Skills |
|------|-------------|---------------|
| **매번 설명** | ✅ 필요함 | ❌ 필요 없음 |
| **일관성** | ⚠️ 매번 다를 수 있음 | ✅ 항상 동일 |
| **효율성** | ⚠️ 시간이 걸림 | ✅ 빠름 |
| **자동화** | ❌ 수동 호출 | ✅ 자동 활성화 |
| **관리** | ⚠️ 복잡함 | ✅ 간단함 |
| **팀 공유** | ⚠️ 힘듦 | ✅ 쉬움 |

### 1.3 Claude Skills가 필요한 경우 vs 불필요한 경우

#### ✅ Claude Skills가 필수인 경우:

**경우 1: 반복되는 작업**
- 매주/매달/매년 반복되는 보고서
- 정기적으로 작성하는 문서
- 자주 발생하는 작업

**경우 2: 정해진 형식**
- 회사 브랜드 가이드라인
- 표준 템플릿이 있는 문서
- 정해진 구조의 산출물

**경우 3: 팀/조직에서 공유**
- 여러 사람이 반복하는 일
- 표준화가 필요한 업무
- 일관성이 중요한 작업

#### ❌ Claude Skills가 불필요한 경우:

**경우 1: 일회성 작업**
- 특정 프로젝트 한 번만 필요
- 새로운 상황에 대한 즉흥적 판단
- 반복될 가능성 없음

**경우 2: 매우 창의적인 작업**
- 새로운 아이디어 브레인스토밍
- 독창적인 콘텐츠 창작
- 규칙을 벗어난 창의력 필요

**경우 3: 구조가 자주 변경**
- 매번 다른 형식 요구
- 일관된 규칙이 없음
- 유동적인 프로세스

### 1.4 Claude Skills의 3가지 핵심 가치

#### 가치 1: 시간 절약
```
보고서 작성에 30분 걸렸던 작업
→ Claude Skills: 2분

월 8회 작업 × 28분 = 224분(3.7시간)
연 96회 작업 × 28분 = 2,688분(44.8시간)

연간 45시간 절약!
```

#### 가치 2: 일관성 보장
```
여러 사람이 작성한 보고서:
- 사람 A: 파란색 헤더, Arial 폰트
- 사람 B: 초록색 헤더, Times New Roman
- 사람 C: 검은색 헤더, Verdana

Claude Skills 사용:
→ 모두 동일한 형식으로 자동 생성
→ 회사 이미지 통일
```

#### 가치 3: 전문성 향상
```
새 직원도 Claude Skills로:
→ 경력 10년 직원과 동일한 수준의 산출물
→ 온보딩 시간 단축
→ 품질 보증
```

## Chapter 3: 환경 설정 - 폴더 구조 만들기

### 3.1 폴더 생성하기 (Windows)

#### Step 1: 적절한 위치 선택

먼저, 스킬을 저장할 폴더를 어디에 만들지 결정합시다.

**추천 위치:**
```
C:\Users\[당신의 이름]\Documents\Claude-Skills
```

또는:
```
바탕화면
```

> **초보자 팁:** 바탕화면이 가장 쉽습니다!

#### Step 2: 폴더 생성 (Windows)

**방법 1: 마우스 사용**

1. 바탕화면 빈 공간에 **우클릭**
   
   ![우클릭 메뉴가 나타남]

2. "새로 만들기" → "폴더" 클릭
   
   ![메뉴에서 "새로 만들기" 위에 마우스 올려놓으면 서브메뉴 표시]

3. 새로 나타난 폴더의 이름을 입력: `my-first-skill`
   
   > **주의:** 절대 스페이스(공백)를 넣지 마세요!
   > ❌ "my first skill" (X)
   > ✅ "my-first-skill" (O)
   > ✅ "myFirstSkill" (O도 됨)

4. **Enter 키** 누르기

완료! 이제 바탕화면에 `my-first-skill` 폴더가 생겼습니다.

#### Step 3: 폴더 열기

생성한 폴더를 **더블클릭**하여 열기

**현재 상태:**
```
my-first-skill/
  (빈 폴더)
```

### 3.2 폴더 생성하기 (Mac)

#### Step 1: Finder 열기

**방법 1: Dock에서**
- 아래쪽 Dock에서 "Finder" 아이콘 클릭

**방법 2: 바탕화면에서**
- 바탕화면 어디든 클릭 후 Cmd + Space 누르기
- "Finder" 입력 후 Enter

#### Step 2: 폴더 생성 (Mac)

**방법 1: 키보드 단축키 (가장 빠름)**

1. 폴더를 만들고 싶은 위치에서 다음을 누르기:
   ```
   Cmd + Shift + N
   ```

2. 폴더 이름 입력: `my-first-skill`

3. **Return 키** 누르기

**방법 2: 메뉴 사용**

1. Finder 상단 메뉴에서 "File" → "New Folder" 클릭

2. 폴더 이름 입력: `my-first-skill`

3. **Return 키** 누르기

#### Step 3: 폴더 열기

생성한 폴더를 **더블클릭**하여 열기

### 3.3 폴더 내부 구조 이해하기

#### 기본 구조: 최소 버전 (지금은 이것만 사용)

```
my-first-skill/
```

#### 완전한 구조: 나중에 확장할 때

```
my-first-skill/

├── examples/                   (선택)
│   ├── good_example.txt
│   └── bad_example.txt
├── scripts/                    (선택)
│   └── generate_report.py
├── templates/                  (선택)
│   └── report_template.docx
└── resources/                  (선택)
    └── logo.png
```

## Chapter 4: SKILL.md 파일 만들기

### 4.1 SKILL.md는 뭔가요?

**SKILL.md = 당신의 스킬의 정체성과 설명서**

이 파일 하나가 Claude에게 다음을 알려줍니다:
- "난 뭔가요?" (이름, 설명)
- "뭘 할 수 있어요?" (기능)
- "어떻게 써요?" (사용 방법)
- "예시 있어요?" (예시)

### 4.2 메모장(또는 TextEdit)에서 SKILL.md 만들기

#### Windows 사용자:

**Step 1: 메모장 열기**

1. 키보드에서 **Windows 키** 누르기
   
2. "메모장" 입력
   
3. "메모장" 앱 클릭
   
   메모장이 열렸습니다!

**Step 2: 내용 복사하기**

아래 내용을 **전체 선택(Ctrl+A)하여 복사**한 후, 메모장에 **붙여넣기(Ctrl+V)**:

```
---
name: weekly-report-skill
description: 지난주의 업무 내용을 표준화된 형식의 주간 보고서로 정리합니다. 진행 중, 완료, 이슈 세 가지 카테고리로 구분하며, 마크다운 형식으로 작성합니다. 매주 월요일이나 금요일에 업무를 정리할 때 사용하세요.
---

# 주간 업무 보고서

## 목표
지난주 업무를 명확하게 정리하여 팀과 공유합니다.

## 형식

### 1️⃣ 진행 중인 업무
- 작업명: ...
- 진도: ...%
- 다음 단계: ...

### 2️⃣ 완료된 업무
- 작업명: ...
- 결과: ...
- 배운 점: ...

### 3️⃣ 이슈 및 문제점
- 문제: ...
- 영향도: ...
- 해결 계획: ...

## 예시

### 진행 중인 업무
- 작업명: 신규 고객 데이터베이스 구축
- 진도: 60%
- 다음 단계: 데이터 검증 및 정제

### 완료된 업무
- 작업명: 분기 마케팅 계획 수립
- 결과: 4가지 캠페인 기획 완료
- 배운 점: 고객 세그먼트 분석이 중요함

### 이슈 및 문제점
- 문제: 외부 API 연동 지연
- 영향도: 높음 (프로젝트 일정에 영향)
- 해결 계획: 담당 회사에 긴급 연락, 우회 방안 검토

## 추가 팁
- 각 항목은 명확하고 구체적으로 작성하세요
- 진도는 백분율로 표현하세요
- 이슈는 즉시 보고하세요
```

**Step 3: 파일로 저장하기**

2. "파일 이름" 입력창에:

3. "저장 위치"를 **내가 만든 폴더** (my-first-skill)로 변경

완료! 이제 파일이 저장되었습니다.

#### Mac 사용자:

**Step 1: TextEdit 열기**

1. **Cmd + Space** 누르기

2. "TextEdit" 입력

3. **Return** 누르기

**Step 2: 포맷 변경**

## Chapter 5: 파일 내용 자세히 이해하기

### 5.1 SKILL.md의 두 부분

SKILL.md는 항상 두 부분으로 나뉩니다:

#### 부분 1: YAML 프론트매터 (위쪽)

```yaml
---
name: weekly-report-skill
description: 지난주의 업무 내용을 표준화된 형식의 주간 보고서로 정리합니다.
---
```

**이 부분은 절대 수정하면 안 됩니다!**

역할:
- `name`: 스킬의 고유 이름 (Claude가 스킬을 인식하는 ID)
- `description`: Claude에게 "이 스킬이 뭘 하는지" 알려주는 설명

#### 부분 2: 마크다운 본문 (아래쪽)

```markdown
# 주간 업무 보고서

## 목표
지난주 업무를 명확하게 정리하여 팀과 공유합니다.

## 형식
...
```

**이 부분은 자유롭게 수정해도 됩니다!**

역할:
- Claude에게 "이 일을 어떻게 해야 하는지" 구체적으로 설명

### 5.2 description 필드 정하기 (가장 중요!)

**description**은 Claude가 스킬을 선택하는 유일한 신호입니다.

따라서 **매우 명확하고 구체적**이어야 합니다.

#### ❌ 나쁜 예:
```
description: 보고서를 만듭니다
```
→ Claude가 언제 이 스킬을 써야 하는지 모름

#### ✅ 좋은 예:
```
description: 지난주의 업무 내용을 표준화된 형식의 주간 보고서로 
정리합니다. 진행 중, 완료, 이슈 세 가지 카테고리로 구분하며, 
마크다운 형식으로 작성합니다. 매주 월요일이나 금요일에 업무를 
정리할 때 사용하세요.
```

**좋은 description의 특징:**

1️⃣ **"뭘 하는가?"** - 구체적인 기능
   - "주간 보고서 작성"
   - "진행 중, 완료, 이슈로 분류"

2️⃣ **"언제 쓰는가?"** - 사용 상황
   - "매주 월요일"
   - "업무를 정리할 때"

3️⃣ **"어떻게 하는가?"** - 방식
   - "마크다운 형식"
   - "세 가지 카테고리로 구분"

### 5.3 나머지 내용 이해하기

#### 제목 섹션
```markdown
# 주간 업무 보고서
```
- `#` 하나 = 가장 큰 제목
- 스킬의 이름을 명확히 표시

#### 목표 섹션
```markdown
## 목표
지난주 업무를 명확하게 정리하여 팀과 공유합니다.
```
- `##` = 두 번째 크기 제목
- 이 스킬의 "왜" 만드는지 설명

#### 형식 섹션
```markdown
## 형식

### 1️⃣ 진행 중인 업무
- 작업명: ...
- 진도: ...%
```
- 실제 사용할 형식을 보여줌
- Claude가 따라할 구체적 기준 제시

#### 예시 섹션
```markdown
## 예시

### 진행 중인 업무
- 작업명: 신규 고객 데이터베이스 구축
```
- 실제 작성된 예시를 보여줌
- 텍스트 설명보다 훨씬 효과적!

## Chapter 6: ZIP 파일 만들기

### 6.1 ZIP 파일이 뭔가요?

**ZIP 파일 = 폴더를 압축한 것**

마치 선물을 상자에 포장하는 것처럼, 폴더 전체를 하나의 파일로 압축합니다.

**왜 ZIP으로 만들어야 하나요?**

Claude.ai에서는 스킬을 업로드할 때 ZIP 파일 형식을 요구합니다. 따라서 우리 폴더를 ZIP으로 압축해야 합니다.

### 6.2 Windows에서 ZIP 파일 만들기

#### 방법 1: 마우스 우클릭 (가장 간단)

**Step 1: 폴더 찾기**

내가 만든 폴더 (my-first-skill)를 찾아서 **우클릭**

**Step 2: 메뉴에서 선택**

```
우클릭 메뉴:
┌─────────────────────────┐
│ 열기                    │
│ 편집                    │
│ 보내기 ─→ 압축          │  ← 여기!
│ 삭제                    │
│ 속성                    │
└─────────────────────────┘
```

"보내기" → "압축" 선택

**Step 3: 완료!**

폴더와 같은 위치에 `my-first-skill.zip` 파일이 생성됩니다!

#### 방법 2: 메뉴 사용

1. 폴더 열기

2. 상단 메뉴에서 **"공유" 탭** 클릭

3. **"압축"** 버튼 클릭

4. ZIP 파일 생성됨

### 6.3 Mac에서 ZIP 파일 만들기

#### 방법 1: 우클릭 (가장 간단)

**Step 1: 폴더 찾기**

Finder에서 내 폴더 (my-first-skill) **우클릭**

**Step 2: 메뉴에서 선택**

```
우클릭 메뉴:
┌──────────────────────────────┐
│ 열기                         │
│ 정보 보기                    │
│ "my-first-skill"(을)를      │
│ 압축                         │  ← 여기!
│ 삭제                         │
└──────────────────────────────┘
```

"'my-first-skill'을를 압축" 선택

**Step 3: 완료!**

`my-first-skill.zip` 파일이 생성됩니다!

#### 방법 2: 키보드 단축키

1. 폴더 선택

2. **Cmd + C** (복사)

3. 같은 폴더에 마우스 우클릭

4. "압축" 선택

### 6.4 ZIP 파일 확인하기

#### 생성되었는지 확인

윈도우/맥 모두:

1. 내 폴더가 있는 위치에서 새로고침 (F5)

2. `my-first-skill.zip` 파일이 보이면 성공!

```
폴더:
├── my-first-skill/          (원본 폴더)
└── my-first-skill.zip       ← 새로 생성됨!
```

#### ZIP 파일 크기 확인

**파일을 우클릭 → 속성 확인**

대부분 1-10KB 정도의 작은 파일입니다.

만약 100MB 이상이면, 뭔가 잘못되었습니다!

## Chapter 9: 스킬 개선하기

### 9.1 스킬이 완벽하지 않을 때

첫 번째 시도에 완벽한 스킬을 만들 사람은 없습니다.

대신, **반복해서 개선**하는 것이 중요합니다.

### 9.2 스킬 수정하는 법

#### Step 1: 파일 열기

내 SKILL.md 파일을 메모장 또는 VS Code에서 열기

#### Step 2: 내용 수정

원하는 부분 변경

예를 들어, description을 더 구체적으로:

**Before:**
```yaml
description: 주간 보고서를 만듭니다
```

**After:**
```yaml
description: 지난주의 업무 진행 상황을 "진행 중", "완료", "이슈" 
세 가지로 구분하여 마크다운 형식의 주간 보고서로 정리합니다. 
매주 월요일 아침이나 금요일 오후에 주간 업무를 정리할 때 사용하세요.
```

#### Step 3: 저장

**메모장:** Ctrl+S

**VS Code:** Cmd+S (Mac) 또는 Ctrl+S (Windows)

#### Step 4: ZIP 파일 다시 만들기

폴더를 다시 ZIP으로 압축

#### Step 5: 재업로드

Claude.ai의 Settings → Skills에서 다시 업로드

```
이전 버전: 자동으로 교체됨
새 버전: 즉시 적용
```

### 9.3 개선할 만한 항목들

#### 1️⃣ Description 개선

스킬이 호출되지 않으면, **description을 더 구체적으로** 작성

```
나쁜 예: "데이터를 분석합니다"
좋은 예: "Excel 파일의 판매 데이터를 분석하여 월별/지역별 매출 
트렌드를 파악하고, 성장률을 계산한 후, 이상치를 탐지합니다. 
매달 첫 주에 지난달 매출 데이터를 분석할 때 사용하세요."
```

#### 2️⃣ 형식 명확화

현재 형식이 너무 복잡하면, **더 간단하게**:

```markdown
## 형식

### 진행 중
- 작업명
- 진도 (%)
- 예정 완료일

### 완료됨
- 작업명
- 결과
- 완료일

### 이슈
- 문제
- 심각도 (높음/중간/낮음)
```

예시 섹션을 **더 구체적으로**:

```markdown
### Input (사용자 요청)
```

## Chapter 11: REFERENCE.md 파일 추가하기

### 11.1 언제 REFERENCE.md가 필요한가?

SKILL.md가 너무 길어지면 (500줄 이상), **일부를 REFERENCE.md로 분리**합니다.

#### 예시:

**SKILL.md가 너무 길 때:**
```markdown
# 주간 업무 보고서

## 개요
...

## 기본 형식
...

## 상세 규칙 (이 부분이 너무 길어짐)
- 규칙 1: ...
- 규칙 2: ...
- [50개의 규칙...]

## 예시
...
```

→ 이런 경우, "상세 규칙"을 REFERENCE.md로 이동

### 11.2 REFERENCE.md 만들기

#### Step 1: 파일 생성

SKILL.md와 같은 폴더에 `REFERENCE.md` 파일 생성

#### Step 2: 내용 이동

SKILL.md에서 상세한 내용을 잘라내어 REFERENCE.md에 붙여넣기

**REFERENCE.md 예시:**
```markdown
# 주간 업무 보고서 - 상세 규칙

## 진행 중 업무의 상세 규칙

### 진도 표기법
- 0-25%: 계획 단계
- 26-50%: 초기 구현
- 51-75%: 마무리 단계
- 76-99%: QA 단계
- 100%: 완료 (완료됨으로 이동)

### 작업명 작성 규칙
- 명사 + 액션 형태: "데이터베이스 구축"
- 너무 길지 않게: 10단어 이내
- 구체적으로: "작업" 대신 "고객관계 시스템 구축"

... [더 많은 규칙]
```

#### Step 3: SKILL.md에서 링크

SKILL.md 본문에 다음을 추가:
```markdown
더 상세한 규칙은 REFERENCE.md를 참조하세요.
```

#### Step 4: ZIP 재생성

새 폴더 구조:
```
my-first-skill/
├── SKILL.md
└── REFERENCE.md

→ 압축 → my-first-skill.zip
```

## Chapter 13: 스킬 버전 관리하기

### 13.1 왜 버전 관리가 필요한가?

스킬을 계속 개선하다 보면, "어떤 버전이 최신인지", "뭐가 바뀌었는지" 헷갈릴 수 있습니다.

따라서 **버전을 명확히 기록**합니다.

### 13.2 버전 표기법

SKILL.md의 첫 부분에 버전 추가:

```yaml
---
name: weekly-report-skill
description: ...
version: 1.0.0
---
```

**버전 표기: X.Y.Z**

- X: 큰 변화 (0 → 1로 증가)
- Y: 기능 추가 (0 → 1로 증가)
- Z: 버그 수정 (0 → 1로 증가)

#### 예시:

```
v1.0.0 (2025-02-01): 초기 버전 출시
  - 기본 형식 정의
  - 3가지 카테고리 (진행중/완료/이슈)

v1.1.0 (2025-02-10): 기능 추가
  - "배운 점" 필드 추가
  - "다음 단계" 필드 추가

v1.1.1 (2025-02-15): 버그 수정
  - description 명확화
  - 예시 추가

v2.0.0 (2025-03-01): 큰 변화
  - 새로운 형식으로 완전 재설계
  - 5가지 카테고리로 확대
```

### 13.3 변경 이력 기록하기

SKILL.md 끝에 "버전 히스토리" 섹션 추가:

```markdown
## 버전 히스토리

### v1.1.0 (2025-02-10)
**추가 사항:**
- "배운 점" 필드 추가
- "다음 단계" 필드 추가

**변경 사항:**
- description 더 구체적으로 수정
- 예시 섹션 확대

### v1.0.0 (2025-02-01)
초기 버전 출시
```

# PART 5: 트러블슈팅 & FAQ

## Chapter 15: 문제 해결

### 15.1 "파일이 열리지 않습니다"

**증상:** SKILL.md 파일을 더블클릭해도 안 열림

**원인:** 파일의 확장자가 잘못됨

**해결:**

1. 파일을 우클릭 → "이름 바꾸기"

2. 이름 확인:
   - ❌ "SKILL" (확장자 없음)
   - ❌ "SKILL.txt" (txt 확장자)
   - ✅ "SKILL.md" (md 확장자)

3. 정정하여 저장

### 15.2 "ZIP 파일이 손상되었습니다"

**증상:** ZIP 파일을 압축 해제할 때 오류

**원인:** 폴더를 압축하는 과정에서 오류 발생

**해결:**

1. 폴더를 다시 확인:
   ```
   my-first-skill/
   └── SKILL.md
   ```

2. 다시 압축하기:
   - 우클릭 → 압축
   - 또는 OS의 기본 압축 프로그램 사용

3. 새 ZIP 파일 업로드

### 15.3 "Claude가 스킬을 인식하지 못합니다"

**증상:** 업로드했는데 스킬이 작동하지 않음

**확인 단계:**

**1️⃣ 활성화 확인**

Settings → Skills에서 **체크박스가 활성화**되어 있는가?

```
☑️ weekly-report-skill    ← 체크되어 있어야 함
```

미체크라면 → 클릭하여 활성화

**2️⃣ Description 확인**

스킬의 description이 **충분히 구체적**인가?

```
❌ 나쁜: "보고서를 만들어줍니다"
✅ 좋은: "주간 업무 현황을 진행중/완료/이슈로 구분한 보고서로 정리합니다. 매주 월요일에 사용하세요."
```

**3️⃣ 테스트 요청 확인**

사용자의 요청이 **스킬과 관련**이 있는가?

```
❌ "이 책 요약해줘" (스킬과 무관)
✅ "지난주 업무 보고서 만들어줘" (스킬과 관련)
```

### 15.4 "설정된 형식을 따르지 않습니다"

**증상:** 스킬은 작동하지만, 형식이 설정과 다름

**원인:** SKILL.md의 지침이 너무 복잡하거나 모호함

**해결:**

SKILL.md의 "형식" 섹션을 더 간단하고 명확하게:

```markdown
## 형식 (Before - 너무 복잡)
진행 중인 업무의 경우, "진도를 백분율로 표현하고, 
다음 단계를 명사구로 작성하며, 예상 완료일을 
ISO 8601 형식으로 기입하고..."

## 형식 (After - 간단하고 명확)
진행 중인 업무:
- 작업명: [내용]
- 진도: [%]
- 다음 단계: [내용]
```

### 15.5 "파일을 찾을 수 없습니다" 오류

**증상:** 업로드 시 "SKILL.md를 찾을 수 없습니다" 오류

**원인:** ZIP 파일 구조가 잘못됨

**확인:**

1. ZIP 파일 압축 해제

2. 구조 확인:

**❌ 잘못된 구조:**
```
SKILL.md          ← 폴더 없이 바로 여기
```

**✅ 올바른 구조:**
```
my-first-skill/
└── SKILL.md
```

**해결:**

1. 내용을 폴더 안으로 이동

2. 폴더 자체를 ZIP으로 압축

3. 재업로드

## Chapter 16: FAQ - 자주 묻는 질문

### Q1: 스킬을 만드는 데 코딩이 필요한가요?

**A:** 아니요! 기본적인 스킬은 마크다운 파일만으로 만들 수 있습니다. 
코딩은 데이터 처리 같은 복잡한 작업을 할 때만 선택사항입니다.

### Q2: 한 개의 스킬로 여러 기능을 할 수 있나요?

**A:** 가능하지만, 권장하지 않습니다. 
하나의 스킬은 하나의 명확한 목적을 가져야 합니다.
대신 **여러 스킬을 조합**하는 것이 좋습니다.

### Q3: 스킬을 수정하면 이전 버전은 어떻게 되나요?

**A:** 새 버전으로 자동 교체됩니다. 
이전 버전을 보관하려면, 버전 히스토리를 SKILL.md에 기록하세요.

### Q4: 스킬이 모든 사용자에게 동일하게 작동하나요?

**A:** 대부분의 경우 네. 하지만 Claude의 모델 버전에 따라 
조금 다를 수 있습니다.

### Q5: 스킬을 삭제할 수 있나요?

**A:** 네. Settings → Skills에서 체크박스를 **해제**하면 비활성화되고, 
삭제 버튼이 있으면 클릭하여 제거할 수 있습니다.

### Q6: 한 번에 여러 스킬을 업로드할 수 있나요?

**A:** 현재는 각 스킬을 따로 업로드해야 합니다. 
하지만 ZIP 파일을 여러 개 만들어서 순서대로 업로드할 수 있습니다.

### Q7: 스킬의 성능을 측정할 수 있나요?

**A:** 직접적으로는 아니지만, 다음으로 확인할 수 있습니다:
- 스킬이 호출되었는가? (Claude의 응답에서 확인)
- 결과의 품질이 좋은가? (주관적 평가)
- 시간이 얼마나 절약되었는가? (before/after 비교)

### Q8: 회사 기밀 정보를 스킬에 포함시켜도 되나요?

**A:** 개인 스킬이면 괜찮지만, 팀과 공유할 때는 민감한 정보를 
빼야 합니다. 예를 들어:
- ❌ 실제 고객 이름과 숫자
- ✅ "[고객명]", "[금액]" 같은 플레이스홀더

### Q9: 스킬을 스스로 학습하거나 개선할 수 있나요?

**A:** 현재는 아니지만, 미래에 가능할 수 있습니다. 
지금은 사용자가 필요에 따라 **수동으로 스킬을 개선**해야 합니다.

### Q10: 무료로 모든 기능을 사용할 수 있나요?

**A:** 기본 스킬 기능은 Claude 계정이 있으면 무료로 사용 가능합니다. 
Claude Plus 구독이 권장되지만, 필수는 아닙니다.

```yaml
# 주간 업무 보고서

## 목표
지난주의 업무를 명확하고 체계적으로 정리하여 팀 리더 또는 
스테이크홀더와 공유합니다.

## 구성 요소

### 1️⃣ 진행 중인 업무 섹션
이 섹션은 아직 완료되지 않은 진행 중인 모든 작업을 나열합니다.

**작성 방식:**
- 작업명: 명확하고 구체적인 이름 (예: "고객 관계 시스템 구축")
- 진도: 0-100% 사이의 숫자
- 다음 단계: 다음 주에 할 일

**예시:**
```
- 작업명: 고객 관계 시스템 구축
  진도: 65%
  다음 단계: 데이터 검증 및 오류 수정
```

### 2️⃣ 완료된 업무 섹션
완료된 모든 작업을 나열합니다.

**작성 방식:**
- 작업명: 완료한 작업의 이름
- 결과: 어떤 산출물이 나왔는가?
- 배운 점: 이 작업에서 배운 교훈

**예시:**
```
- 작업명: Q1 마케팅 계획 수립
  결과: 3가지 캠페인 기획서 제출
  배운 점: 고객 데이터 분석이 기획에 매우 중요함
```

### 3️⃣ 이슈 및 문제점 섹션
진행 중이거나 새로 발생한 문제를 나열합니다.

**작성 방식:**
- 문제: 구체적인 문제 설명
- 영향도: "높음", "중간", "낮음" 중 선택
- 해결 계획: 어떻게 해결할 것인가?

**예시:**
```
- 문제: 외부 API 응답이 느려짐
  영향도: 높음
  해결 계획: API 제공자와 협의하거나, 우회 방안 검토
```

## 작성 시 주의 사항
1. 가능한 구체적이고 정량적으로 작성
2. 진도는 반드시 백분율로 표현
3. 문제는 최대한 빨리 보고
4. 각 항목은 1-3줄 이내로 간결하게

## 제출 빈도
- 권장: 매주 금요일 오후
- 또는: 매주 월요일 아침
```

```yaml
# 고객 이메일 템플릿

## 이메일 구조

### 인사 및 감사
고객의 요청에 감사하는 따뜻한 인사로 시작

### 상황 설명 (거절의 경우)
거절하는 이유를 **객관적이고 사실 기반**으로 설명

### 대체 안 제시
다른 방법이나 일정을 제안

### 마무리 및 연락처
계속된 협력을 기대하는 긍정적인 종료

## 예시

### 거절 이메일
```
안녕하세요 [고객명]님,

[제안/요청]에 관해 문의해주셔서 감사합니다.

현재 저희 팀은 진행 중인 [프로젝트명] 때문에 
추가 프로젝트를 수용하기 어려운 상황입니다. 
그러나 [월/날짜]부터는 가능할 것으로 예상됩니다.

대신, [대체 안]을 제안드립니다:
1. [대체 방안 1]
2. [대체 방안 2]

다시 한번 문의해 주셔서 감사하며, 
[예상 시기]에 뵙기를 기대합니다.

[서명]
```

### 제안 이메일
```
안녕하세요 [고객명]님,

[상황 설명]

이와 관련하여 저희가 제안할 수 있는 솔루션이 있습니다:

1. [제안사항 1] - 장점: [...]
2. [제안사항 2] - 장점: [...]
3. [제안사항 3] - 장점: [...]

자세한 내용은 첨부 파일을 참조해 주세요.

편한 시간에 미팅을 통해 더 자세히 논의하고 싶습니다.
[날짜/시간] 중에 가능하신가요?

감사합니다.

[서명]
```
```

## Chapter 1: Claude Skills란 무엇인가? {#chapter-1}

### 1-1. Claude Skills의 정의

Claude Skills를 가장 쉽게 설명하면 다음과 같습니다:

**Claude에게 주는 '업무 매뉴얼' 또는 '전문 지식 패키지'**

실제 업무 상황을 생각해보세요. 신입사원을 채용했을 때 어떻게 교육하나요?

1. 말로 설명한다
2. 매뉴얼을 준다
3. 과거 사례를 보여준다
4. 작업 체크리스트를 제공한다
5. 참고 자료를 준비한다

Claude Skills는 정확히 이 모든 과정을 AI에게 적용하는 것입니다. Claude를 여러분의 업무 방식에 맞춰진 '전문가'로 만드는 것이죠.

### 1-2. 왜 Claude Skills가 필요한가?

만약 Claude Skills 없이 매번 같은 작업을 한다면:

```
사용자: "우리 회사 스타일로 보고서 써줘"
Claude: 일반적인 형식의 보고서 생성

다음 날
사용자: "우리 회사 스타일로 또 다른 보고서 써줘"
Claude: 또 다른 형식으로 생성 (이전과 다를 수 있음)

또 다음 날
사용자: "같은 형식으로..."
Claude: "제가 지난 형식을 기억하지 못합니다..."
```

**Claude Skills를 사용하면:**

```
처음: Skills 설정 (1회만)
사용자: "우리 회사 스타일로 보고서 써줘"
Claude: Skills를 자동으로 로드 → 정확한 형식의 보고서 생성

다음 날
사용자: "보고서 만들어줘"
Claude: 같은 Skills 자동 로드 → 동일한 형식의 보고서 생성

계속 같은 일관성 유지!
```

### 1-3. Claude Skills의 핵심 특징

**특징 1: 자동 적용**
- 스킬을 설정하면 관련된 요청을 할 때 Claude가 자동으로 적용합니다
- "이 스킬을 써줘"라고 말할 필요가 없습니다

**특징 2: 여러 플랫폼 호환성**
- Claude Desktop (윈도우/맥)
- Claude.ai 웹사이트
- Claude API
- Claude Code
- 모든 플랫폼에서 동일하게 작동합니다

**특징 3: 재사용 가능**
- 한 번 만든 스킬을 팀원들과 공유 가능
- 다른 프로젝트에도 활용 가능

**특징 4: 확장 가능**
- 텍스트 지침만 있어도 작동
- 점점 고도화하면서 데이터, 코드, 예시 추가 가능

## Chapter 2: Claude Skills로 할 수 있는 일 {#chapter-2}

### 2-1. 기본적인 활용 분야

#### 마케팅 분야
- **상품 소개 문구 생성**: 회사의 톤앤매너와 스타일 규칙을 적용하여 일관된 마케팅 카피 생성
- **A/B 테스트 카피**: 여러 버전의 광고 문구를 자동으로 생성
- **이메일 템플릿**: 고객별 맞춤 이메일의 기본 틀 자동 생성
- **소셜 미디어 콘텐츠**: 플랫폼별 특성에 맞는 포스트 작성

#### 문서 작성
- **보고서 자동화**: 주간/월간 보고서를 정해진 형식대로 생성
- **회의록 정리**: 음성/텍스트를 회사 표준 회의록으로 변환
- **이력서 검수**: 이력서를 표준 형식으로 변환하고 문법 검사
- **제안서 작성**: 고객별 맞춤 제안서 초안 자동 생성

#### 개발/기술
- **API 문서 생성**: 코드에서 자동으로 API 문서 초안 작성
- **코드 리뷰**: 팀의 코드 리뷰 기준에 따라 자동 검토
- **기술 문서 작성**: 일관된 포맷의 기술 문서 생성
- **버그 리포트 정리**: 난잡한 버그 리포트를 표준 형식으로 변환

#### 영업/고객관리
- **제안 이메일**: 고객별로 맞춤화된 제안 이메일 자동 생성
- **고객 응대 문구**: 다양한 고객 요청에 대한 표준 응대 문구
- **계약서 검토**: 회사 계약 정책에 따른 자동 검토
- **CRM 데이터 정리**: 고객 정보를 표준 형식으로 입력

#### 인사/HR
- **면접 질문 생성**: 공고와 지원자 정보에 맞춘 맞춤 면접 질문
- **채용공고 작성**: 일관된 형식의 채용공고 자동 생성
- **온보딩 자료**: 신입사원 교육용 자료 자동 생성
- **직원 평가**: 평가 기준에 따른 자동 평가안 작성

## Chapter 3: 기본 개념 이해하기 {#chapter-3}

### 3-1. Claude Skills의 구조

Claude Skills는 **폴더 구조**를 기반으로 합니다. 가장 단순한 형태부터 복잡한 형태까지 다음과 같이 구성됩니다.

#### 최소 구조 (가장 간단함)
```
my-skill/
└── SKILL.md
```

이것만으로도 기본적인 스킬이 작동합니다.

#### 기본 구조
```
my-skill/
├── SKILL.md          # 필수: 스킬의 모든 지침
└── REFERENCE.md      # 선택: 추가 참고 자료
```

#### 완전한 구조
```
my-skill/
├── SKILL.md              # 필수: 스킬의 정의와 지침
├── REFERENCE.md          # 선택: 상세한 참고 자료
├── examples/             # 선택: 입출력 예시
│   ├── good_input.txt
│   ├── bad_input.txt
│   ├── good_output.txt
│   └── bad_output.txt
├── scripts/              # 선택: 실행 가능한 코드
│   ├── process.py
│   ├── analyze.py
│   └── requirements.txt
└── resources/            # 선택: 이미지, 템플릿, 설정
    ├── logo.png
    ├── template.docx
    └── config.json
```

### 3-2. 각 파일의 역할

| 파일 | 역할 | 중요도 | 필수 여부 |
|------|------|--------|---------|
| **SKILL.md** | 스킬의 정의와 지침 | ⭐⭐⭐⭐⭐ | 필수 |
| **REFERENCE.md** | 추가 참고 자료 | ⭐⭐⭐ | 선택 |
| **examples/** | 입출력 예시 | ⭐⭐⭐⭐ | 선택 |
| **scripts/** | 실행 가능한 코드 | ⭐⭐⭐⭐⭐ | 선택 |
| **resources/** | 보조 자료 | ⭐⭐⭐ | 선택 |

#### SKILL.md - 스킬의 핵심
- **역할**: 스킬이 어떤 것인지, 어떻게 작동하는지 정의
- **포함 내용**: 메타데이터, 지침, 예시, 가이드라인
- **중요성**: ⭐⭐⭐⭐⭐ (가장 중요)
- **필수 여부**: 반드시 필요

#### REFERENCE.md - 추가 정보
- **역할**: SKILL.md에는 너무 많은 추가 자료 제공
- **포함 내용**: 상세 가이드, 기술 참고자료, 법적 문구, 템플릿 레이아웃
- **중요성**: ⭐⭐⭐ (있으면 좋음)
- **필수 여부**: 선택 (정보가 많으면 권장)

#### examples/ - 실제 예시
- **역할**: Claude가 원하는 스타일을 이해하도록 "보여주기"
- **포함 내용**: 좋은 입력 예시, 나쁜 입력 예시, 원하는 출력, 피해야 할 출력
- **중요성**: ⭐⭐⭐⭐ (이해도를 높이는 데 매우 효과적)
- **필수 여부**: 선택 (복잡한 작업일수록 강력)

#### scripts/ - 자동화 코드
- **역할**: Claude가 실제로 실행할 수 있는 파이썬 코드
- **포함 내용**: 데이터 분석, 파일 생성, API 호출 등의 파이썬 스크립트
- **중요성**: ⭐⭐⭐⭐⭐ (자동화의 핵심)
- **필수 여부**: 선택 (자동화가 필요할 때만)

#### resources/ - 보조 자료
- **역할**: 이미지, 문서, 설정 파일 등 참고 자료 보관
- **포함 내용**: 로고 파일, 워드 템플릿, 설정 파일, 데이터 파일
- **중요성**: ⭐⭐⭐ (필요할 때만)
- **필수 여부**: 선택

### 3-3. Claude가 Skills를 사용하는 과정

Claude는 다음과 같은 순서로 Skills를 활용합니다:

**단계 1: 메타데이터 읽기 (항상)**
- 사용자의 요청이 들어오면, Claude는 모든 활성화된 스킬의 `name`과 `description`을 읽습니다
- "이 요청에 맞는 스킬이 있나?" 판단합니다
- 이것이 매우 빨리 진행되어 사용자는 지연을 느끼지 못합니다

**단계 2: 스킬 선택**
- 가장 관련성 높은 스킬을 선택합니다
- 여러 스킬이 관련되면 가장 적합한 것부터 로드합니다

**단계 3: 스킬 파일 로드 (선택적)**
- 선택된 스킬의 `SKILL.md` 파일을 읽습니다
- 해당 작업에 필요한 지침을 모두 이해합니다

**단계 4: 추가 정보 로드 (필요시)**
- 더 상세한 정보가 필요하면 `REFERENCE.md`를 읽습니다
- 예시를 확인하려면 `examples/` 파일을 읽습니다
- 코드를 실행해야 하면 `scripts/` 파일을 로드합니다

**단계 5: 작업 실행**
- 스킬의 지침을 따라 사용자의 요청을 처리합니다
- 필요하면 `scripts/`의 코드를 실행합니다

**단계 6: 결과 제공**
- 완성된 결과물을 사용자에게 제공합니다
- 파일이 생성되었으면 그 파일의 경로를 알려줍니다

### 3-4. Description의 중요성

**스킬의 가장 중요한 부분은 `description` 필드입니다.**

Claude가 100개 이상의 스킬 중에서 여러분의 요청에 맞는 스킬을 찾아내는 것은 이 `description`을 읽고 판단하기 때문입니다.

**나쁜 Description 예:**
```
description: 문서를 만듭니다.
```
→ 언제 써야 할지 모릅니다. 너무 일반적입니다.

**좋은 Description 예:**
```
description: 팀의 주간 업무 현황을 '진행 중', '완료', '이슈' 세 가지 항목으로 구분하여 정해진 형식의 마크다운 보고서를 작성합니다. 각 항목은 글머리 기호로 정리되며, 마지막에는 다음 주 예정사항을 추가합니다.
```
→ 정확히 언제, 어떤 상황에서, 어떻게 작동하는지 명확합니다.

## Chapter 4: 폴더와 파일 구성하기 (완전 입문자용) {#chapter-4}

### 4-1. 작업 환경 준비하기

이 섹션에서는 완전 초보자를 위해 윈도우와 맥 모두에서 진행할 수 있도록 설명합니다.

#### 필요한 도구
1. **텍스트 에디터** (파일을 만들고 편집하기 위해)
   - 추천: VS Code (무료, 강력함) https://code.visualstudio.com/
   - 또는: 메모장 (윈도우 기본 제공)
   - 또는: TextEdit (맥 기본 제공)

2. **압축 해제 도구**
   - 윈도우: 기본 제공 (마우스 우클릭 > 압축 해제)
   - 맥: 기본 제공 (압축 파일을 더블클릭)

3. **Claude Desktop 또는 웹사이트**
   - Claude Desktop: https://claude.ai/download
   - 웹사이트: https://claude.ai

### 4-2. 스킬 폴더 위치 이해하기

Claude가 스킬을 찾는 위치는 정해져 있습니다. 반드시 이 위치에 폴더를 만들어야 합니다.

#### Windows 사용자
```
C:\Users\[사용자명]\.claude\skills\
```

예: `C:\Users\john\.claude\skills\`

#### Mac 사용자
```
/Users/[사용자명]/.claude/skills/
```

예: `/Users/john/.claude/skills/`

**숨김 폴더 찾는 법 (처음에는 보이지 않음)**

**Windows (File Explorer):**
1. 주소 표시줄에 `%USERPROFILE%\.claude\skills` 붙여넣기
2. Enter 누르기
3. 폴더가 없으면 직접 만들기 (오른쪽 마우스 > 새 폴더)

**Mac (Finder):**
1. Command + Shift + . (마침표)를 누르면 숨김 파일 표시
2. 홈 폴더에서 `.claude/skills/` 폴더 찾기
3. 폴더가 없으면 Terminal에서 만들기:
```bash
mkdir -p ~/.claude/skills
```

### 4-3. 첫 번째 스킬 폴더 만들기

실습용으로 간단한 "주간 보고서" 스킬을 만들어 봅시다.

#### 단계 1: 폴더 생성

**Windows (File Explorer):**
1. `C:\Users\[사용자명]\.claude\skills\` 폴더 열기
2. 오른쪽 마우스 클릭 > 새로 만들기 > 폴더
3. 폴더 이름: `weekly-report-skill` (또는 한글로 `주간보고서`)

## 개요

이 스킬은 팀원들의 주간 업무 현황을 체계적으로 정리하여 회사 표준 형식의 보고서를 생성합니다.

### 6-1. SKILL.md 파일 상세 분석

#### 메타데이터 블록

```markdown

- 스킬을 업데이트할 때 증가

### 6-2. Description 완벽하게 쓰기

Description은 스킬의 성공을 결정하는 가장 중요한 요소입니다.

#### Description 작성의 3가지 요소

**요소 1: WHAT (무엇을)**
```
❌ 나쁜 예: "보고서를 만듭니다"
✅ 좋은 예: "마크다운 형식의 주간 업무 보고서를"
```

**요소 2: WHEN (언제/어떤 상황에)**
```
❌ 나쁜 예: "필요할 때"
✅ 좋은 예: "팀원의 주간 업무 내용을 정리할 때"
```

**요소 3: HOW (어떻게)**
```
❌ 나쁜 예: "작성합니다"
✅ 좋은 예: "'진행 중', '완료', '이슈' 섹션으로 구분하여 생성합니다"
```

#### 최종 예시

**최악의 Description:**
```
description: 문서를 만듭니다.
```
→ 언제 써야 할지, 어떤 문서인지 알 수 없습니다.

**나쁜 Description:**
```
description: 보고서 작성 스킬입니다.
```
→ 어떤 보고서인지 여전히 불명확합니다.

**괜찮은 Description:**
```
description: 주간 업무 보고서를 생성합니다.
```
→ 어떤 보고서인지는 알지만, 어떤 형식인지 모릅니다.

**좋은 Description:**
```
description: 팀원의 주간 업무를 '진행 중', '완료', '이슈' 섹션으로 구분하여 마크다운 보고서로 생성합니다.
```
→ 대부분의 정보가 명확합니다.

**최고의 Description:**
```
description: 팀원의 주간 업무 현황을 입력받아 회사 표준 형식('진행 중', '완료', '이슈')의 마크다운 주간 보고서를 자동으로 생성합니다. 각 섹션은 명확한 이모지 기호와 함께 구분되며, 마지막에는 다음 주 계획을 포함합니다.
```
→ 언제, 무엇을, 어떻게 할지 모두 명확합니다.

# 마케팅 카피 생성 가이드

## 개요

이 스킬은 신제품 또는 서비스를 마케팅할 때 필요한 다양한 스타일의 카피를 자동으로 생성합니다.

## 지침

### 1단계: 입력 정보 수집

다음 정보를 사용자로부터 받으세요:
- 제품명
- 제품의 주요 특징 (최대 3개)
- 타겟 고객층
- 제품 가격 (선택사항)

### 2단계: 3가지 카피 스타일 생성

#### 스타일 1: 감정 중심
- 고객의 감정과 욕구에 호소
- "당신의 삶이 어떻게 변할까?"라는 질문형 시작
- 긍정적이고 꿈 같은 톤

#### 스타일 2: 논리 중심
- 데이터와 사실에 기반
- 제품의 구체적인 이점 나열
- "이유는..." 로 시작하는 설득 구조

#### 스타일 3: 긴급성 중심
- 한정된 시간/수량 강조
- 행동 촉구 (Call-to-Action)
- 직접적이고 명령조 톤

## 예시

**입력:**
```

# 고객 응대 표준 응답 가이드

## 개요

고객 만족도를 높이기 위해 모든 응답은 다음 4단계를 따릅니다:
1. **공감**: 고객의 상황을 이해하고 공감
2. **원인 파악**: 문제의 원인을 명확히 함
3. **해결책**: 구체적인 해결 방안 제시
4. **감사와 약속**: 고객 감사 및 추가 도움 약속

## 응답 구조

### 1단계: 공감 문구
```
"[고객명]님, [상황]로 인해 불편을 드려 정말 죄송합니다."
```

### 2단계: 원인 파악
```
"저희 확인 결과, [원인]에서 문제가 발생한 것으로 파악됩니다."
```

### 3단계: 해결책
```
"이를 해결하기 위해 [해결책]을 시행하겠습니다."
또는
"고객님께서는 [방법]으로 문제를 해결하실 수 있습니다."
```

### 4단계: 감사와 약속
```
"귀중한 피드백을 주셔서 감사합니다. 앞으로 이런 일이 없도록 노력하겠습니다."
```

- 책임감 있는 톤 (변명보다는 해결책)
- 명확하고 구체적인 설명

import pandas as pd
import matplotlib.pyplot as plt
from datetime import datetime

def analyze_sales(csv_file):
    """판매 데이터 분석 메인 함수"""
    
    # 데이터 로드
    df = pd.read_csv(csv_file)
    df['date'] = pd.to_datetime(df['date'])
    
    # 기본 통계
    total_sales = df['total_sales'].sum()
    num_transactions = len(df)
    avg_daily_sales = df.groupby('date')['total_sales'].sum().mean()
    
    # 제품별 분석
    product_sales = df.groupby('product_name')['total_sales'].sum().sort_values(ascending=False)
    top_5_products = product_sales.head(5)
    
    # 보고서 생성
    report = f"""# 월간 판매 데이터 분석 보고서

**분석 기간:** {df['date'].min().strftime('%Y년 %m월')}
**보고서 작성일:** {datetime.now().strftime('%Y년 %m월 %d일')}

## 📊 매출 개요

- **총 매출:** {total_sales:,.0f}원
- **거래 건수:** {num_transactions}건
- **일평균 매출:** {avg_daily_sales:,.0f}원

## 🏆 상위 5개 제품

"""
    
    for idx, (product, sales) in enumerate(top_5_products.items(), 1):
        percentage = (sales / total_sales) * 100
        report += f"{idx}. **{product}**: {sales:,.0f}원 ({percentage:.1f}%)\n"
    
    # 차트 생성
    plt.figure(figsize=(12, 6))
    daily_sales = df.groupby('date')['total_sales'].sum()
    daily_sales.plot(kind='line', marker='o')
    plt.title('일별 판매액 추이')
    plt.xlabel('날짜')
    plt.ylabel('판매액 (원)')
    plt.grid(True, alpha=0.3)
    plt.tight_layout()
    plt.savefig('sales_chart.png', dpi=150)
    plt.close()
    
    # 보고서 저장
    with open('sales_report.md', 'w', encoding='utf-8') as f:
        f.write(report)
    
    print("✅ 분석 완료!")
    print(f"📄 보고서: sales_report.md")
    print(f"📈 차트: sales_chart.png")

if __name__ == "__main__":
    analyze_sales('sales_data.csv')

#### 문제 2: "스킬이 보이지만 작동하지 않음"

description: 팀의 주간 업무 현황을 입력받아 '진행 중', '완료', '이슈' 세션으로 구분하는 마크다운 보고서를 생성합니다.

올바른 형식:
```markdown
---
name: 스킬명
description: 설명

---
```

잘못된 형식:
```markdown
--
name: 스킬명

--
```

**원인 3: 마크다운 문법 오류**

1. 목록 마크다운: `- ` (대시 뒤에 공백) 사용

3. 코드: ` ``` ` (백틱 3개) 사용
4. 들여쓰기: 탭 대신 스페이스 4개 사용

## Chapter 9: 고급 활용법 {#chapter-9}

### 9-1. 여러 스킬 함께 사용하기

하나의 스킬만으로는 복잡한 작업을 할 수 없을 때가 있습니다. 여러 스킬을 함께 사용하면 강력한 자동화가 가능합니다.

#### 예: 고객 피드백 → 분석 → 응대 → 보고

**스킬 구성:**

1. **스킬 1: 고객 피드백 분류**
   - 불만, 칭찬, 개선제안으로 분류
   
2. **스킬 2: 감정 분석**
   - 긍정/부정/중립 판단
   
3. **스킬 3: 표준 응대 문구 생성**
   - 분류 결과에 맞는 응대 생성
   
4. **스킬 4: 피드백 분석 보고서**
   - 모든 피드백을 집계하여 보고

**작동 흐름:**
```
사용자: "고객 피드백 5개 처리해줄 수 있어?"
  ↓
Claude: 스킬 1 자동 호출 (분류)
  ↓
Claude: 스킬 2 자동 호출 (감정 분석)
  ↓
Claude: 스킬 3 자동 호출 (응대 문구)
  ↓
Claude: 스킬 4 자동 호출 (보고서)
  ↓
결과: 분류된 피드백 + 감정 분석 + 표준 응대 + 보고서
```

### 9-2. 스킬과 데이터 파일 함께 사용하기

스킬 폴더에 데이터 파일을 넣으면 Claude가 참조할 수 있습니다.

#### 예: 가격 책정 스킬 + 가격표 데이터

**폴더 구조:**
```
pricing-skill/
├── SKILL.md
├── REFERENCE.md
└── resources/
    └── price_list.csv
```

**SKILL.md에서:**
```markdown
## 가격 정보

resources/price_list.csv 파일을 참조하여 다음과 같이 가격을 결정합니다:

- 기본 상품: $10
- 프리미엄 상품: $20
- 엔터프라이즈: 맞춤 가격

자세한 가격표는 resources/price_list.csv를 참조하세요.
```

### 9-3. 스킬 버전 관리

스킬을 개선할 때마다 버전을 올립니다.

```markdown
---
name: 스킬명
description: 설명
version: 1.0.0  ← 첫 번째 릴리스

version: 1.1.0  ← 버그 수정 (Minor)
version: 1.2.0  ← 기능 추가 (Minor)
version: 2.0.0  ← 큰 변화 (Major)
---
```

**버전 번호 규칙:**
- Major.Minor.Patch
- Major: 큰 변화 (하위 호환성 깨짐)
- Minor: 기능 추가 (하위 호환성 유지)
- Patch: 버그 수정 (하위 호환성 유지)

### 9-4. 스킬과 Claude Code 함께 사용하기

Claude Code를 사용하면 더 복잡한 자동화가 가능합니다.

예: 엑셀 파일 자동 생성
```python
# 스킬의 scripts/create_excel.py
import pandas as pd
from openpyxl.styles import Font, PatternFill

def create_report_excel(data_dict):
    """데이터를 엑셀로 변환"""
    df = pd.DataFrame(data_dict)
    
    # 엑셀 저장
    excel_file = 'report.xlsx'
    df.to_excel(excel_file, index=False)
    
    # 스타일 적용 (선택사항)
    from openpyxl import load_workbook
    wb = load_workbook(excel_file)
    ws = wb.active
    
    # 헤더 스타일
    for cell in ws[1]:
        cell.font = Font(bold=True, color="FFFFFF")
        cell.fill = PatternFill(start_color="4472C4", end_color="4472C4", fill_type="solid")
    
    wb.save(excel_file)
    print(f"✅ 엑셀 파일 생성: {excel_file}")
```

### Q5: 스킬이 오작동하면?

**1단계: 개별 테스트**
- 스킬만 활성화하고 다른 스킬은 비활성화

- Description이 너무 광범위하면 좁혀서 수정
- 더 구체적인 키워드 추가

**4단계: 메타데이터 재확인**
- SKILL.md 파일의 메타데이터 부분 확인
- 형식이 정확한지 확인

### Q6: 스킬에서 외부 API를 호출할 수 있나요?

**가능합니다!**

```python
# scripts/call_api.py
import requests

### Q7: 스킬과 MCP(Model Context Protocol)의 차이는?

**스킬:**
- 지식/지침 제공
- 형식, 스타일, 프로세스 정의
- 텍스트/데이터 기반
- 예: "보고서 형식", "마케팅 가이드"

**MCP:**
- 도구/능력 제공
- 외부 시스템과 연동
- 코드 실행, API 호출
- 예: "날씨 조회", "이메일 발송", "데이터베이스 쿼리"

### Q8: 스킬을 비활성화해도 파일이 삭제되나요?

**아니요, 안전합니다!**
- 비활성화: 사용하지 않기만 함 (파일은 보관)
- 필요하면 다시 활성화 가능
- 파일을 완전히 삭제하려면 수동으로 삭제해야 함

### Q9: 스킬은 최신 Claude 모델에서만 작동하나요?

**모든 Claude 모델에서 작동:**
- Claude 3.5 Sonnet
- Claude 3 Opus/Sonnet/Haiku
- 구형 모델도 대부분 지원

**팁:**
- 더 강력한 모델일수록 복잡한 스킬도 잘 처리
- Haiku 모델: 더 명확한 지침 필요
- Opus 모델: 복잡한 지침도 잘 이해

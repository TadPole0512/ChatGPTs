| # | 공식 | 주요 변수 |
| -- | | |
| 1 | 천재적 통찰력 | GI = (관찰 × 연결 × 패턴 × 통합) / (가정 + 편견) |
| 2 | 다차원 | 5가지 차원: 시간-공간-추상화-인과관계-위계 |
| 3 | 창의적 연결 | 직접 - 간접 - 역설 - 은유 - 시스템 연결 |
| 4 | 문제 재정의 | 180° 뒤집기 - 스케일 이동 - 개념 이동 - 도메인 이동
| 5 | 혁신 솔루션 | 결합-차용-제약→우위-역발상-시스템 재설계
| 6 | 통찰력 증폭 | 5가지 이유 - 만약에 - 어떻게 할 수 있을까?
| 7 <사고의 진화> 학습-경험-성찰-교환-실패 학습
| 8 | 복잡성 해결 | 분해 → 관계 매핑 → 레버리지 포인트 |
| 9 | 직관적 도약 | (침묵 × 경험 × 신뢰) / (논리 × 합리화) |
| 10 | 통합적 지혜 | (지식 + 이해 + 지혜 + 연민 + 행동) × 겸손 × 윤리 |


질문 해석 우선순위
1. 사전 브리핑 확인: RealGrid 버전, 프레임워크, 데이터 규모, 성능 목표 우선 파악
2. 문제 유형 분류: 초기 설정/성능 이슈/UI 커스터마이징/데이터 연동/디버깅으로 구분
3. 비동기 처리 주의: 모든 RealGrid 관련 질문에서 async/callback 충돌 가능성 검토

코드 제공 규칙

// ✅ 항상 포함해야 할 요소

- HTTP 상태 확인 코드 (API 연동 시)

🧠 D. RealGrid 문제 해결 방법론

디버깅 3단계 프로세스

// 1단계: 초기화 순서 검증
console.log('1. DataProvider 생성:', dataProvider);
console.log('2. GridView 생성:', gridView);
console.log('3. setDataSource 호출 완료');

// 2단계: 데이터 상태 확인
console.log('fields 정의:', dataProvider.getFields());
console.log('현재 행 수:', dataProvider.getRowCount());
console.log('첫 번째 행 데이터:', dataProvider.getJsonRow(0));

// 3단계: 이벤트 추적
gridView.onCurrentRowChanged = function(grid, oldRow, newRow) {
  console.log('행 변경:', oldRow, '→', newRow);
  console.log('선택 데이터:', grid.getValues(newRow));
};


비동기 충돌 해결 패턴

// ❌ 피해야 할 패턴
function init() {
  setupGrid();     // 비동기
  loadCombo();     // 비동기 - 충돌 가능!
  fetchData();     // 비동기
}

// ✅ 권장 패턴
async function init() {
  await setupGrid();
  await loadCombo();
  await fetchData();
}
// 또는
function init() {
  setupGrid(() => {
    loadCombo(() => {
      fetchData();
    });
  });
}

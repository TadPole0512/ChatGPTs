목표 Google Slides와 Gemini in Slides, Nano Banana Pro 및 자동화 도구(Apps Script/Slides API)를 활용해 즉시 업무에 적용할 수 있는 실무 매뉴얼을 제공한다.

Nano Banana Pro는 Gemini 3 Pro 기반으로 텍스트가 잘 읽히는 고품질 이미지를 여러 언어로 생성하며【895433497974522†L283-L301】, 모든 AI 이미지에는 SynthID 워터마크가 삽입되고 무료/Pro 요금제에서는 눈에 띄는 워터마크가 유지된다【895433497974522†L520-L538】.

1. Google Slides 핵심 조작 (기본기)

1.1 테마·레이아웃·테마 빌더

  • 개념: 테마(Theme)는 전체 프레젠테이션의 색상·폰트·배경 스타일을 정의하고, 레이아웃(Layout)은 각 슬라이드에 배치될 요소(제목/본문/이미지)의 틀이다. Theme Builder(옛 Master Slide)를 이용하면 전체 덱의 디자인을 일괄 수정할 수 있다. Computerworld 기사에 따르면 테마 빌더를 열려면 상단 메뉴에서 “View > Theme builder”를 선택한다【433063014431033†L270-L307】.
  • 언제 사용? 회사 브랜드 가이드에 맞춘 슬라이드를 만들거나 여러 슬라이드의 디자인을 일괄 변경할 때.
  • 어떻게?
    1. Google Slides에서 프레젠테이션을 열고 View > Theme builder 선택【433063014431033†L270-L307】.
    2. 왼쪽 ‘LAYOUTS’ 목록에서 수정할 레이아웃을 클릭한다【433063014431033†L270-L307】.
    3. 불필요한 텍스트나 요소를 선택하여 Delete로 제거하고, 상단 서식 도구를 이용해 도형·이미지·텍스트 상자를 추가한다【433063014431033†L276-L287】.
    4. 객체를 드래그하여 위치와 크기를 조정하고 색상·배경을 수정한다【433063014431033†L286-L294】.
    5. 레이아웃 이름을 클릭해 고유한 이름으로 변경한 후 X 버튼으로 Theme Builder를 종료한다【433063014431033†L296-L307】.
  • 주의사항: 변경한 레이아웃은 현재 프레젠테이션에만 적용된다. 다른 파일에 적용하려면 새 테마로 다운로드하거나 템플릿으로 저장하여 가져와야 한다. 공동편집자가 있는 경우 변경사항이 즉시 반영된다.
  • 대체 경로: 레이아웃 대신 슬라이드 삽입 시 Layout 메뉴에서 원하는 레이아웃을 직접 선택하거나, 템플릿 갤러리에서 미리 디자인된 레이아웃을 사용할 수 있다【529246492776942†L105-L107】.

✅ Steps 1. 메뉴 : View > Theme builder 선택 2. 왼쪽 레이아웃 목록에서 수정할 항목 클릭 3. 요소 삭제·추가·재배치 4. 새 이름 지정 후 닫기 5. 새 레이아웃을 슬라이드에 적용 (Layout 메뉴)

1.2 텍스트·도형·이미지와 정렬

  • 개념: 텍스트 상자, 도형, 이미지 등을 삽입하여 정보를 시각화한다. 정렬과 간격 조절을 통해 일관된 디자인을 유지한다.
  • 실무 팁:
    • 삽입 : 상단 Insert 메뉴에서 Text box/Shape/Image 선택. 이미지는 드라이브 업로드 또는 URL 붙여넣기 가능.
    • 그리드 & 가이드 : View > Snap to 옵션에서 Guides와 Grid를 활성화하여 요소를 균등하게 배치.
    • 정렬 : 여러 객체를 선택 후 Arrange > Align에서 중앙, 좌측, 상단 등 선택.
    • 서식 복사 : 원하는 스타일을 적용한 객체를 선택하고 Paint format 도구를 클릭한 뒤 다른 객체를 클릭하면 같은 스타일을 적용.
    • 도형 결합 : 여러 도형을 선택한 후 Arrange > Merge shapes를 사용하여 복잡한 도형을 만들 수 있다.
  • 주의사항: 이미지 파일은 가능한 16:9 비율로 준비하고 용량을 최적화한다. Nano Banana Pro로 생성한 이미지에는 워터마크가 포함될 수 있으므로 필요한 경우 화면을 자르거나 AI Ultra 플랜을 사용해 제거한다【895433497974522†L520-L538】.
  • 대체 경로: Microsoft PowerPoint에서 작업 후 Google Slides로 업로드하면 텍스트와 도형이 거의 유지되나, 일부 고급 애니메이션과 폰트는 호환되지 않는다.

✅ Steps 1. Insert > Text box/Shape/Image로 요소 삽입 2. View > Guides/Grid로 배치 기준선 활성화 3. 여러 객체 선택 후 Arrange > Align으로 정렬 4. 페인트 브러시(서식 복사)로 스타일 복사 5. 필요 시 Arrange > Merge shapes 활용

1.3 차트·표·다이어그램 구성

  • 개념: 데이터 시각화를 위해 차트와 표를 삽입하며, 다이어그램은 플로우·프로세스를 설명한다.
  • 실무 팁:

3. Gemini in Slides 활용

3.1 기능과 시작 방법

Gemini in Slides는 AI 이미지 생성, 슬라이드 생성, 텍스트 요약/재작성, Drive 파일 참조 기능을 제공한다【477443691059783†L20-L29】. 기능을 사용하려면 슬라이드를 열고 상단 우측에 있는 Ask Gemini 버튼(스파크 아이콘)을 클릭한다【477443691059783†L45-L48】. 사이드 패널이 열리면 다음을 수행할 수 있다:

  • 이미지 생성 – “Generate an image” 또는 “Suggest images for this slide”라고 입력하면 선택된 슬라이드 내용을 분석해 이미지 후보를 보여준다【477443691059783†L100-L114】. 원하는 이미지를 선택해 Insert를 클릭하면 워터마크가 포함된 상태로 삽입된다. AI 모델로는 2025년 11월부터 Nano Banana Pro가 기본으로 사용되며, 인포그래픽과 “Beautify this slide” 기능을 지원한다【591886253299689†L90-L114】.
  • 슬라이드 생성 – “Create a slide about @파일명”과 같이 입력하면 현재 테마에 맞는 새 슬라이드를 생성한다【477443691059783†L121-L143】. “Retry”를 클릭하면 다른 버전을 받을 수 있다.
  • 요약 및 텍스트 생성 – 현재 프레젠테이션을 요약하거나 새로운 내용을 작성하도록 지시할 수 있다. 예: “Summarize this presentation in 3 sentences”【477443691059783†L147-L160】.
  • Drive/Gmail 참조 – @파일명 또는 @이메일을 입력해 Drive 문서나 이메일 내용을 참고자료로 삼아 슬라이드를 생성한다【477443691059783†L28-L29】.
  • 변형/추가 기능 – 사이드 패널에서 Expand 버튼을 눌러 창을 키우거나 Clear history로 이전 대화를 삭제할 수 있다【477443691059783†L70-L80】.

3.2 좋은 입력 규칙

  1. 명확한 목표 제시 – “3분 내 투자 제안서 요약”처럼 결과물의 형태와 길이를 구체적으로 명시한다.
  2. 컨텍스트 제공 – 관련 문서(@파일)와 키워드를 함께 제공하여 모델이 참고할 정보를 늘린다.
  3. 톤/스타일 지시 – “친근하지만 전문적인 톤으로 작성”처럼 어조와 형식을 지정한다.
  4. 반복 활용 – 결과가 기대와 다르면 Retry나 Edit prompt를 사용해 수정한다【477443691059783†L139-L145】.
  5. 출력 검토 – AI 가 생성한 이미지와 텍스트는 반드시 사실 및 브랜드 가이드 준수 여부를 검토하고 필요한 수정 후 삽입한다.

3.3 실패 패턴 및 수정법

  • 톤 불일치: 모델이 지나치게 캐주얼하거나 공식적인 문체를 출력할 경우 prompt에 “~한 어조로 작성해줘”라고 명확히 지시하거나, 생성된 텍스트를 수동으로 수정한다.
  • 정보 누락: Drive 문서 링크나 데이터를 함께 제공하지 않으면 요약이 피상적일 수 있다. 가능한 자료를 @로 참조하고 중요 요소를 나열한다.
  • 레이아웃 깨짐: AI가 생성한 슬라이드가 현재 테마와 조화를 이루지 못할 수 있다. 삽입 후 Theme builder를 통해 서식을 수정하거나, 필요한 경우 “Beautify this slide” 기능을 사용해 다시 생성한다【591886253299689†L90-L114】.
  • 워터마크 문제: 무료 및 Pro 플랜에서는 이미지에 워터마크가 남으므로 공식 발표에서는 적합하지 않을 수 있다. Ultra 플랜을 사용하거나 이미지 편집으로 워터마크 부분을 자른다【895433497974522†L520-L538】.

4. Nano Banana Pro 통합 작업

4.1 정의와 정책

Nano Banana Pro는 Gemini 3 Pro를 기반으로 한 이미지 생성·편집 모델로서, 텍스트가 명확히 읽히는 고해상도 이미지를 다양한 언어로 생성한다【895433497974522†L283-L301】. 이전 세대(Nano Banana 2.5 Flash)에 비해 현실 세계 지식과 검색 기반 정보 활용이 향상되어 정확한 인포그래픽과 다이어그램을 만들어 준다【895433497974522†L338-L347】. 최대 14개의 이미지를 혼합하고 최대 5명의 인물 얼굴 일관성을 유지할 수 있으며【895433497974522†L419-L425】, 4K 해상도까지 지원한다【895433497974522†L296-L300】. 모든 이미지에는 SynthID라는 디지털 워터마크가 삽입되고, 무료·Pro 요금제에는 시각적 워터마크가 포함된다【895433497974522†L520-L538】. 워터마크를 제거하려면 Google AI Ultra 또는 Google AI Studio에서 생성해야 한다.

4.2 슬라이드용 이미지 요구사항

  • 비율: Google Slides는 기본 16:9 화면을 사용하므로 Nano Banana Pro 프롬프트에 16:9 aspect ratio를 명시한다.
  • 텍스트: 텍스트 렌더링이 정확하며, 한글/중국어/일본어 등 다양한 언어로 설명을 삽입할 수 있다【895433497974522†L363-L373】. 제목과 본문은 분리하여 짧게 작성한다.
  • 여백과 대비: 슬라이드 가장자리와 이미지 내용 간 충분한 여백을 두고, 배경과 텍스트 색상 대비를 확보한다.
  • 브랜드 컬러: 프롬프트에 Hex 값이나 브랜드 색상을 명시해 일관성을 유지한다. 로고 삽입이 필요하면 AI 모델 대신 수동으로 추가한다.
  • 파일 형식: PNG 또는 JPG가 일반적이며, 인포그래픽은 SVG로 출력하면 크기 조정 시 품질을 유지할 수 있다.

4.3 워크플로우 및 주의사항

  1. 프롬프트 작성 – 위 표의 템플릿을 참고하여 Goal, Inputs, Style, Constraints, Output Spec 등을 명확히 작성한다.
  2. Gemini in Slides 사용 – Ask Gemini 패널에서 프롬프트를 입력하고 이미지를 생성한 후 Insert를 클릭한다. 단, 60일 무료 사용 한도 이후에는 계정 당 사용량 제한이 적용될 수 있다【591886253299689†L140-L144】.
  3. 품질 확인 – 삽입된 이미지의 텍스트가 선명하고 브랜드 컬러에 맞는지 확인한다. 필요하면 Nano Banana Pro의 편집 기능(D 타입 프롬프트)을 사용해 수정한다.
  4. 워터마크 처리 – 프리 플랜에서는 워터마크가 남으므로 주제에 따라 잘 보이지 않는 위치로 배치하거나, AI Ultra를 구독하여 제거한다【895433497974522†L520-L538】.
  5. 저작권 – Nano Banana Pro 생성물은 상업적 사용이 가능하지만, 내부 가이드에 따라 저작권 표시나 AI 사용 사실을 명시해야 할 수 있다.

5. Slides 자동화 – Apps Script & Slides API

5.1 Apps Script 기본 (Slides Service)

Slides Service는 Apps Script 내장 서비스로서 프레젠테이션을 생성·수정할 수 있으며, 객체를 추가하고 텍스트를 변경하는 등 기본적인 작업을 지원한다【675894132430037†L1327-L1333】. 주요 메서드는 SlidesApp.openById(), Presentation.getSlides(), Slide.insertShape() 등이다. 아래는 예시 코드이다.

예제 1 – 프레젠테이션 생성 및 슬라이드 추가 (Apps Script)
function createDeck() {
  // 프레젠테이션 생성
  const pres = SlidesApp.create('자동 생성 예제');
  const slide = pres.getSlides()[0];
  // 첫 슬라이드에 제목과 부제목 삽입
  slide.insertTextBox('보고서 제목', 50, 50, 400, 50);
  slide.insertTextBox('작성일: 2025-12-12', 50, 120, 400, 30);
  // 새로운 슬라이드 추가 (타이틀 + 두 칼럼 레이아웃)
  const newSlide = pres.appendSlide(SlidesApp.PredefinedLayout.TITLE_AND_TWO_COLUMNS);
  newSlide.getPlaceholder(SlidesApp.PlaceholderType.TITLE).asShape().getText().setText('데이터 요약');
  newSlide.getPlaceholder(SlidesApp.PlaceholderType.BODY).asShape().getText().setText('여기에 요약 내용을 입력하세요.');
}

위 스크립트는 간단한 프레젠테이션을 생성한 후 첫 슬라이드에 텍스트 박스를 삽입하고, 두 번째 슬라이드를 추가한다. 주의: 스크립트 실행 시 권한 요청이 나타나며, Slides에 수정 권한이 있어야 한다.

예제 2 – 스프레드시트 데이터를 슬라이드로 변환 (Apps Script)
function sheetToSlides() {
  const ss = SpreadsheetApp.openById('시트ID');
  const sheet = ss.getSheetByName('데이터');
  const data = sheet.getDataRange().getValues();
  const pres = SlidesApp.create('자동화 리포트');
  data.forEach((row, idx) => {
    if (idx === 0) return; // 헤더 건너뜀
    const slide = pres.appendSlide(SlidesApp.PredefinedLayout.TITLE_AND_BODY);
    slide.getPlaceholder(SlidesApp.PlaceholderType.TITLE).asShape().getText().setText(row[0]);
    slide.getPlaceholder(SlidesApp.PlaceholderType.BODY).asShape().getText().setText(row.slice(1).join('\n'));
  });
}

해당 함수는 Google Sheets에서 데이터를 읽어 각 행을 개별 슬라이드로 생성한다. 데이터 구조가 복잡한 경우 템플릿 슬라이드를 복제하여 배치한다.

5.2 Slides API와 batchUpdate

Slides API의 presentations.batchUpdate 메서드는 여러 업데이트 요청을 한 번에 전송하며, 요청 중 하나라도 실패하면 전체가 적용되지 않는다【222791837349504†L373-L391】. 이는 협업 환경에서 원자성을 보장한다. API 호출은 서비스 계정 또는 OAuth 2.0을 통해 인증해야 하며, 프로젝트에 Slides API를 활성화해야 한다.

예제 3 – Node.js에서 슬라이드 일괄 생성 (TypeScript)
import {google} from 'googleapis';
import {v4 as uuidv4} from 'uuid';

async function generateSlides() {
  const auth = new google.auth.GoogleAuth({
    scopes: ['https://www.googleapis.com/auth/presentations', 'https://www.googleapis.com/auth/drive'],
  });
  const slides = google.slides({version: 'v1', auth});
  const presentation = await slides.presentations.create({requestBody: {title: 'API 생성 예제'}});
  const presId = presentation.data.presentationId as string;
  const slideId = uuidv4();
  const requests = [
    {
      createSlide: {
        objectId: slideId,
        insertionIndex: 1,
        slideLayoutReference: {predefinedLayout: 'TITLE_AND_TWO_COLUMNS'},
      },
    },
    {
      insertText: {
        objectId: slideId,
        text: 'API로 생성된 슬라이드',
        insertionIndex: 0,
      },
    },
  ];
  await slides.presentations.batchUpdate({presentationId: presId, requestBody: {requests}});
  console.log(`Created presentation: ${presId}`);
}
generateSlides().catch(console.error);

이 코드는 Google Slides API를 사용해 새 프레젠테이션을 만들고 두 개의 요청을 포함하는 batchUpdate를 실행한다. 첫 요청은 슬라이드를 생성하고, 두 번째 요청은 텍스트를 삽입한다. 요청 배열을 한 번에 보내야 원자성을 확보할 수 있으며, 동일한 objectId를 중복 사용하지 않도록 주의한다.

예제 4 – 에러 처리 및 재시도 패턴
async function safeBatchUpdate(slides, presId, requests) {
  try {
    await slides.presentations.batchUpdate({presentationId: presId, requestBody: {requests}});
  } catch (error) {
    console.error('Batch update failed:', error);
    // 쿼터 또는 권한 오류일 수 있으므로 1초 후 재시도
    await new Promise(r => setTimeout(r, 1000));
    await slides.presentations.batchUpdate({presentationId: presId, requestBody: {requests}});
  }
}

재시도 로직을 추가하면 네트워크 오류나 일시적 쿼터 제한 시 안정성을 높일 수 있다. 하지만 너무 자주 호출하면 쿼터를 초과할 수 있으므로 요청을 배치하고 지수백오프(exponential backoff)를 적용한다.

5.3 워크플로우: 데이터 → 슬라이드 자동 생성

  1. 데이터 준비 – Google Sheets에 표 형태로 데이터를 저장한다. 첫 행은 헤더, 나머지는 개별 슬라이드의 콘텐츠다.
  2. Apps Script 작성 – 스프레드시트의 데이터를 읽어 Slides API 호출을 준비하는 스크립트를 작성한다. doPost(e) 함수를 만들어 Webhook 형태로 호출할 수도 있다.
  3. Slides API 호출 – Apps Script에서 Advanced Slides Service를 활성화하고 Slides.Presentations.batchUpdate()를 호출한다【900419576519041†L1309-L1323】. 여러 요청을 배열에 담아 한 번에 전송한다.
  4. 권한 처리 – 서비스 계정을 사용하는 경우 Slides 파일에 해당 계정을 편집자로 공유하고, OAuth 승인을 받아야 한다.
  5. 결과 확인 – 생성된 프레젠테이션 링크를 반환하거나 이메일로 공유한다. 필요 시 Nano Banana Pro로 생성한 이미지들을 API 요청의 CreateImageRequest에 포함시켜 완성도를 높인다.

📚 실행 절차 (How to Use)

초급: 기본 슬라이드 제작

  1. 새 프레젠테이션 생성 – Google Drive에서 New > Google Slides를 선택하거나 slides.new를 입력한다.
  2. 테마 선택 – 오른쪽 사이드바에서 새로운 Design elements 패널을 열어 템플릿과 빌딩블록을 선택한다【529246492776942†L87-L100】. 빌딩블록 라이브러리에는 아젠다, 인용문, 통계 등 기본 요소가 포함되어 있다【529246492776942†L94-L101】.
  3. 콘텐츠 삽입 – 텍스트, 이미지, 도형, 차트, 표, 다이어그램을 삽입한다. 그리드와 가이드를 활성화하여 정렬한다.
  4. 발표 설정 – Q&A 링크를 준비하고 Presenter View에서 노트 및 타이머를 설정한다【284837408948019†L61-L66】.
  5. 공유 및 내보내기 – 협업자를 초대하고 PDF/PPTX로 내보내는 방법을 익힌다.

중급: Gemini in Slides 활용

  1. 슬라이드를 열고 우측 상단 Ask Gemini 버튼을 클릭한다.
  2. 프롬프트 박스에 “추상적이고 현대적인 배경 이미지 생성 – 브랜드 색상 #004080 사용, 16:9”와 같이 입력한다. 생성된 이미지 중 원하는 것을 선택하여 삽입한다【477443691059783†L100-L114】.
  3. “@[보고서 링크] 요약해서 새 슬라이드를 만들어줘”와 같이 Drive 문서를 참조하여 자동으로 슬라이드를 생성한다【477443691059783†L121-L143】.
  4. 생성 결과를 검토하고 필요 시 Retry 또는 Edit prompt로 수정한다【477443691059783†L139-L145】.
  5. Nano Banana Pro 프롬프트 템플릿을 이용해 표지·인포그래픽·아이콘을 반복 생성하여 일관된 스타일을 유지한다.

고급: 자동화와 API

  1. Apps Script 프로젝트 생성 – script.google.com에서 새 프로젝트를 만들고 Slides API 및 Drive API 권한을 요청한다.
  2. Advanced Slides Service 활성화 – 에디터 상단 메뉴에서 서비스 추가를 선택하고 Slides API를 활성화한다【900419576519041†L1309-L1326】.
  3. 데이터 읽기 – 스프레드시트 또는 외부 JSON 데이터를 가져와 슬라이드에 삽입할 콘텐츠를 준비한다.
  4. batchUpdate 요청 구성 – 슬라이드 생성, 텍스트 삽입, 이미지 삽입 요청을 배열로 만들어 한 번에 전송한다. 요청 중 하나라도 오류가 있으면 전체가 롤백되므로 입력 값을 철저히 검증한다【222791837349504†L373-L391】.
  5. 배포 및 스케줄링 – 정기적으로 보고서를 생성해야 한다면 Apps Script 트리거(시간 기반)를 설정하거나 웹 앱으로 배포하여 외부에서 호출할 수 있도록 한다.

💡 추가 개선 (Optional Tips)

카드 1 – 업무 자동화 & 통합

  • Sheets–Slides 자동화: Apps Script에서 Google Sheets 데이터를 읽어 슬라이드를 자동으로 생성하는 함수(sheetToSlides())를 정기 트리거로 실행하면 보고서 작성 시간을 크게 줄일 수 있다.
  • Docs→Slides 변환: Google Docs 문서의 구조적 내용을 슬라이드로 변환하는 스크립트를 작성하여 회의록이나 보고서를 손쉽게 프레젠테이션으로 만들 수 있다.
  • Drive 메타데이터 사용: Slides API와 Drive API를 결합해 파일 목록과 권한을 관리하고, 슬라이드를 회사 Intranet에 자동으로 업로드하는 프로세스를 구축한다.

카드 2 – 브랜드 템플릿 관리

  • 테마 파일 저장: 회사 브랜드 템플릿을 .potx 또는 .pptx로 저장한 후 Google Slides에서 불러와 Theme builder를 통해 내장 테마로 활용한다.
  • 전사 배포: Google Drive에서 폴더 권한을 설정해 모든 직원이 템플릿을 복사해 사용할 수 있도록 한다.
  • 버전 업데이트 관리: 주요 브랜드 요소가 변경될 때마다 기존 템플릿을 업데이트하고 변경 이력을 기록한다.

카드 3 – 발표자 스킬 향상

  • 발표 리허설: Presenter View의 타이머와 노트를 활용하여 연습하고, Q&A 기능을 통해 예상 질문을 수집한다.
  • 애니메이션 최소화: 중요한 내용에만 애니메이션을 사용하여 집중도를 높이고, 접근성을 위해 수동 전환 옵션을 제공한다.
  • 음성 및 시각 보조: 녹화 기능으로 자신의 발표를 확인하고 발음과 속도를 개선한다. 필요시 캡션 기능을 이용해 청각 장애가 있는 청중을 지원한다.

카드 4 – 도구 조합 팁

  • Gemini + Nano Banana Pro: Gemini in Slides로 기본 슬라이드를 생성한 뒤 Nano Banana Pro 템플릿을 활용해 이미지와 인포그래픽을 삽입하면 일관성 있는 AI 프레젠테이션을 빠르게 만들 수 있다.
  • Slides + Vids: 2025년 출시된 Google Vids와 결합해 슬라이드에서 영상을 생성하고 Nano Banana Pro로 썸네일을 만든다【591886253299689†L127-L133】.
  • Apps Script + Vertex AI: Vertex AI의 Nano Banana Pro API를 사용해 Apps Script 내에서 직접 이미지를 생성하고 Slides API로 삽입하는 워크플로우를 구축한다 (고급). 이 때 서비스 계정을 사용해 권한을 위임하고 쿼터 관리를 한다.

    - 함수와 내부 구현 로직
    - 각 라인의 목적 또는 의도
    - 전체 소스를 생략하지 말고 바로 소스를 이용할 수 있게 출력



주석 정리

    - 줄 전체가 주석 처리된 코드는 제거하되 설명 주석은 유지합니다.
	- 주석으로 시작하더라도 코딩을 주석 처리한 경우가 아닌 설명이나 로직을 설명한 주석은 지우지 말고 줄 시작이 주석으로 시작하지 않는 줄은 주석을 지우지 말고 남겨줘.




소스 정리

    - 프로그래밍 언어를 자동 인식합니다.
	- 소스를 디자인패턴과 리팩토링을 적용하여 로직 흐름에 따라 순차적으로 배치해줘
	주고 남겨진 소스에 설명이 필요한 부분은 주석을 달아줘.


	- 로직상 사용하지 않거나 불필요한 소스는 주석처리를 하고 왜 주석 처리했는지 자세하게 알려줘


	- 소스 정리할 때 누락되거나 추가되어야 할 소스가 있을 때는 주석으로 자세하고 상세한 설명해주고, 소스를 만들어주거나 예제 소스를 주석으로 생성해줘.
	- 개선이 필요한 부분은 `[수정 필요]`, `[추가 필요]` 등으로 강조해서 왜 수정이나 추가가 필요한지 자세한 설명과 함께 기존 소스를 주석 처리하고 수정된 소스를 만들어줘
	- 삭제를 해야할 소스에는 강조된 표시와 삭제 이유 자세히 알려주고, 삭제 코드는 주석처리해줘.



    - 누락된 로직이 있다면 주석으로 설명하고 예시 코드를 삽입합니다.
    - 예외처리가 부족한 부분에는 상세한 메시지와 함께 예외 로직을 추가합니다.
    - 리팩토링이 필요한 코드는 기존 소스를 주석 처리하고 새로운 개선 버전을 작성합니다.
    - 전체 소스를 생략하지 말고 바로 전체 소스를 이용할 수 있게 출력


주석 달기

    - 함수 내부 구현 로직 설명
    - 각 라인의 목적 또는 의도 설명
    - 파일 상단에 전체 로직 흐름을 `[STEP 1]`, `[STEP 2]`... 형식으로 요약합니다.
    - 파일 하단에는 상단 흐름 요약과 연결된 전체 로직 상세 설명을 추가합니다.
    - 함수마다 Java 스타일의 문서 주석(JavaDoc/XML)을 작성합니다.
    - 함수 내부의 각 단계(변수 선언, 조건문, 반복문, API 호출 등)에 대해 의미 있는 설명 주석을 상세하게 추가합니다.
    - 함수 내부의 소스의 기능이나 작업에 의미 있는 설명 주석을 상세하게 추가합니다.
    - 불필요한 코드나 사용되지 않는 코드는 [삭제 필요]와 이유를 함께 설명한 뒤 주석 처리합니다.
    - 들여쓰기, 정렬, 가독성 포맷을 일관되게 적용합니다.

    - 완성된 소스를 전체 출력하며, 주석 기반으로 전체 구조를 쉽게 이해할 수 있게 합니다.

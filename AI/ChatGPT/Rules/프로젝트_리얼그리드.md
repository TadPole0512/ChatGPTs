1) 이 문서가 추가로 고정하는 것(요지 5줄)

    - 질문 입력 형식은 “Pre-brief → Questions → 증빙(코드/데이터)”로 고정하고, 번들 단위로 처리합니다. 
    - GUI-우선 출력(카드·표·접기·배지)를 기본 레이아웃으로 채택합니다. 
    - RealGrid는 “버전·라이선스·Provider↔GridView 연결·Excel(jszip)”을 최소 재현 세트로 필수화합니다.   
    - 웹개발 답변은 보안/성능/배포 체크리스트를 동반해야 하며, 가능한 한 “배포 가능한 전체 소스”를 제공합니다. 
    - 아이디어 확장 과업은 ‘생각 공식’ 2개를 명시하고 블렌드 분석 → 아이디어 10+를 산출합니다(필요 시). 



2) 번들링·흐름 규칙(프로젝트 적용)

    - 문장 단위 분해 → Pre-brief/질문 분류 → 순서/우선순위 부여 → 번들 단위 답변. 긴 출력은 “개요→세부” 분할 출력하되 내용 누락 없음.  
    - 반복/충돌 관리: 3회 이상 유사 질문은 형식 전환(표/카드), 5회 이상은 중간 합본 제시 후 재개 권유. 최신 기준 정렬. 



3) GUI-우선 출력 규격(요약)

    - 시각 계층: 헤더(아이콘+제목) → 표/다이어그램 → 하이라이트 박스 → 본문. 
    - 대시보드/카드·진행바·상태표시·액션센터를 기본 컴포넌트로 사용(예시 서식 내장).  
    - 표는 3–6열 권장, 7열↑는 분할. 접기(세부근거) 기본. 



4) RealGrid 전용 지침(질문·재현·정책)

4-1. 질문 서두(Pre-brief) 필수 항목

    - 프레임워크, RealGrid 정확 버전, 로딩 방식(CDN/NPM), 라이선스 적용 방식, 데이터 계약(fields/columns), 정렬/필터 클라/서버 위임, Excel 요구, 성능 목표(p95), 재현코드/샘플데이터 포함.  

4-2. 최소 재현 세트(둘 중 하나 반드시 첨부)

    - CDN 단일 HTML: setLicenseKey → Provider/GridView 생성·연결 → fields/columns → sorting/filtering 옵션 → setRows → exportGrid(+jszip).   
    - React 단일 컴포넌트: 동일 순서 + `npm i realgrid jszip`. 언마운트 정리 포함.  

4-3. 서버 위임 기준·매핑

    - 임계값 가이드: 5만 행↑부터 정렬/필터/페이징은 서버 위임 검토. 그리드 상태 → API 파라미터 매핑 테이블로 일관 처리.  

4-4. Excel 내보내기 규칙

    - jszip 필수 + `exportGrid({type:'excel', target:'local|remote', fileName, showProgress})` 명시. 원격 모드 사용 시 서버 URL 제공.  

4-5. 디버그 패키지(버그 리포트)

    - index.html/단일 컴포넌트 + data.json(50–200행) + 정확 버전/로딩/라이선스 + 재현 단계 + 콘솔·네트워크 캡처.  



5) 웹개발(백/프론트/배포) 답변 표준

    - 성능 체크리스트(프런트: 코드 스플릿·이미지 최적화·CDN·SW 캐싱·Critical CSS / 백엔드: 커넥션 풀·100ms 내 쿼리·다단 캐시·레이트리밋 / DB: 인덱스·N+1 제거·리드레플리카·파티셔닝). 
    - 보안 기본: JWT·CORS·보안헤더·입력검증·세션/토큰 저장소 정책을 코드와 함께 제시. 
    - 컨테이너/배포: 멀티-스테이지·비root·헬스체크·K8s(HPA/프로브/TLS)·CI/CD(테스트→품질→보안→빌드→롤링배포). 
    - 모니터링: Prometheus/Grafana, 경보(에러율>5%, P95>500ms 등) 명시. 

> 코드 제공 원칙(프로젝트 특화): 사용자가 요구한 대로 “부분이 아닌 전체 소스” 우선. 불가피한 부분 소스 시, 파일 경로·줄 범위·패치(diff)를 반드시 함께 표기.



6) 아이디어·설계 과업 시(선택 규칙)

    - 생각 공식 2개 선택 사전 선언 → 블렌드 분석(≥1,500자) → 아이디어 10+ 산출(총 ≥3,000자). 상황별 추천 매트릭스를 활용.  



7) 출력·검증 부가 규칙(프로젝트 차별화)

    - 언어/형식: 한국어 기본, 표/카드/접기 우선, 코드는 실행 가능·환경/입출력·예상결과·에러케이스 명시.  
    - 링크/숫자: URL 유효성(200/리디렉션 최종), 시간 표기는 `YYYY-MM-DD HH:mm (KST)`. 



8) 번들 스켈레톤(붙여넣기용, 압축형)

    - [Pre-brief 요약] 배경/범위/환경(버전·라이선스)/데이터계약/정책(클라vs서버)/목표(p95)
    - [Questions] Q1…Qn
    - [Answer – 카드 요약] 핵심 3포인트 · 표(3~5지표) · TODO 체크박스
    - <접기: 근거/대안/부작용/테스트케이스/로그>
    - [Wrap-up] 결정/변경점/다음액션

(상기 스켈레톤은 Rules v2.2 템플릿을 GUI-우선으로 재구성함)

    - 연결 순서: setLicenseKey → Provider/GridView 생성/연결 → fields/columns → sorting/filtering → setRows → exportGrid(jszip). 코드 첨부 필수.
